## Problem Statement

Flatten a binary tree into a "linked list" where the linked list is represented using the right child pointer of each node and the left child pointer is always null.

<Aside type="tip" title="What to notice first">
- The flattened tree uses only right pointers (like a single-linked list).
- Pre-order traversal order is preserved: root, left subtree, right subtree.
- Left pointers are always set to null in the result.
</Aside>

### Examples

| Input | Output | Order |
| --- | --- | --- |
| `[1,2,5,3,4,null,6]` | `[1,null,2,null,3,null,4,null,5,null,6]` | Pre-order: 1→2→3→4→5→6 |

## Constraints

- The number of nodes is in the range `[0, 2000]`.
- `-100 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- File system hierarchies and directory structures
- DOM tree manipulation in web browsers
- XML/JSON parsing and configuration management
- Organizational hierarchies and permission trees
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Pre-Order DFS | O(n) | O(h) | Intuitive; finds rightmost before attaching |
| Post-Order DFS | O(n) | O(h) | Elegant reverse traversal; track previous |

## Approach 1: Pre-Order DFS

<ApproachTag tag="recommended" />

Recursively flatten left and right subtrees, then find the rightmost node of the flattened left subtree and attach the right subtree to it.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="yellow" timeNote="Visit each node once" spaceNote="Recursion depth" />

### Pseudocode

```plaintext
function flatten(root):
    if not root: return
    
    flatten(root.left)
    flatten(root.right)
    
    if root.left:
        rightmost = root.left
        while rightmost.right:
            rightmost = rightmost.right
        
        rightmost.right = root.right
        root.right = root.left
        root.left = null
```

## Approach 2: Post-Order DFS with Previous Tracking

<ApproachTag tag="optimal" />

Traverse in post-order (right → left → node), tracking the previous node. Attach previous to current's right and nullify left.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="yellow" timeNote="Single pass" spaceNote="Recursion depth" />

### Pseudocode

```plaintext
function flatten(root):
    prev = null
    
    function dfs(node):
        if not node: return
        
        dfs(node.right)
        dfs(node.left)
        
        node.right = prev
        node.left = null
        prev = node
    
    dfs(root)
```

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Pre-Order:** More intuitive, easier to visualize where to attach subtrees.
- **Post-Order:** More elegant, single pass with previous tracking; demonstrates understanding of reverse traversal.
- **Difference:** Pre-order requires finding rightmost; post-order does it implicitly via reverse order.

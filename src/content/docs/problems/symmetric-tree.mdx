## Problem Statement

Given the `root` of a binary tree, check whether it is a mirror of itself (i.e., symmetric around its center).

A tree is **symmetric** if its structure mirrors across the center and all corresponding node values are equal.

<Aside type="tip" title="What to notice first">
- A symmetric tree has identical structure on both sides of the root.
- Each left subtree must mirror the right subtree: left's left child pairs with right's right child, and left's right child pairs with right's left child.
- Only the structure and values matter—the positions in memory do not.
</Aside>

### Examples

| Tree | Symmetric? | Explanation |
| --- | --- | --- |
| `[1, 2, 2, 3, 4, 4, 3]` | ✓ Yes | Mirror structure: 2 on left mirrors 2 on right, children are symmetric pairs |
| `[1, 2, 2, null, 3, null, 3]` | ✗ No | Both children 3 are on the right side, not mirrored |
| `[1]` | ✓ Yes | Single node is symmetric by definition |

## Constraints

- The number of nodes in the tree is in the range `[1, 1000]`.
- `-100 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **GUI layout validation** — check if a window's component layout is balanced and symmetric.
- **Configuration validation** — verify that left and right configurations in a system match (e.g., disk mirror verification).
- **Data structure validation** — ensure tree-based index or search structures maintain balance.
- **Molecular structure analysis** — determine if molecular graphs have mirror symmetry.
- **Image processing** — detect symmetry in image pyramids or multi-scale representations.
</Aside>

## Complexity Comparison

| Approach | Time | Space | When Best |
| --- | --- | --- | --- |
| DFS Recursive | O(n) | O(h) | General case, simple logic, or shallow trees |
| BFS Iterative | O(n) | O(w) | Avoid stack overflow, wide trees, or level-wise analysis |

\* h = tree height; w = max width at any level

## Which approach should you learn first?

- Pick **DFS Recursive** if you are comfortable with recursion and want the most concise solution.
- Pick **BFS Iterative** if you prefer iterative logic or expect very deep trees.

## Approach 1: DFS Recursive (Recommended)

<ApproachTag tag="recommended" />

Check if two subtrees are mirrors of each other. For each pair of nodes at corresponding positions, verify that their values are equal and their children form mirror pairs.

The key insight is to **compare the left and right subtrees as mirror images**: the left's left child should match the right's right child, and the left's right child should match the right's left child.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="yellow" timeNote="Visit each node once" spaceNote="Call stack depth" />

### Step-by-step Example

**Input:** `root = [1, 2, 2, 3, 4, 4, 3]`

```
    1
   / \
  2   2     <- These are mirrors if their children match in mirror fashion
 / \ / \
3  4 4  3   <- Symmetric pairs verified
```

| Step | Compare | Action |
| --- | --- | --- |
| 1 | `isMirror(root, root)` start | Compare root's left and right subtrees |
| 2 | Both nodes exist, values match (2 == 2) | Check children in mirror order |
| 3 | Left's left (3) mirrors right's right (3) ✓ | Left's right (4) mirrors right's left (4) ✓ |
| 4 | All leaf comparisons match | Return `true` |

### Pseudocode

```plaintext
function isSymmetric(root):
    return isMirror(root, root)

function isMirror(left, right):
    if left is null and right is null:
        return true
    if left is null or right is null:
        return false
    if left.val != right.val:
        return false
    return isMirror(left.left, right.right) and
           isMirror(left.right, right.left)
```

## Approach 2: BFS Iterative

<ApproachTag tag="interview" />

Use a queue to process node pairs level by level. Instead of recursion, explicitly maintain a queue of (left, right) pairs to compare, checking the mirror property iteratively.

This approach avoids recursion entirely and may perform better with very deep or unbalanced trees (reducing stack overflow risk).

<ComplexityBadges time="O(n)" space="O(w)" timeColor="green" spaceColor="yellow" timeNote="Visit each node once" spaceNote="Queue width at widest level" />

### Step-by-step Example

**Input:** `root = [1, 2, 2, 3, 4, 4, 3]`

After initialization: `queue = [(root.left, root.right)]` = `[(node 2, node 2)]`

| Step | Queue | Action |
| --- | --- | --- |
| 1 | `[(2, 2)]` | Values match, enqueue children in mirror order |
| 2 | `[(3, 3), (4, 4)]` | Both pairs have matching values, enqueue their children |
| 3 | `[]` (all leaf children are null pairs) | All null pairs continue, queue empties |
| 4 | Done | Return `true` (no mismatches found) |

### Pseudocode

```plaintext
function isSymmetricBFS(root):
    if root is null:
        return true
    queue = [(root.left, root.right)]
    while queue is not empty:
        left, right = queue.dequeue()
        if left is null and right is null:
            continue
        if left is null or right is null:
            return false
        if left.val != right.val:
            return false
        queue.enqueue((left.left, right.right))
        queue.enqueue((left.right, right.left))
    return true
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Wrong mirror pairing:** Comparing `left.left` with `right.left` instead of `right.right`. The mirror property requires cross-pairing.
- **Forgetting null checks:** A tree with missing children on only one side is not symmetric. Always check for null mismatches.
- **Modifying the tree:** Some solutions try to destructively mark visited nodes. This breaks the original tree. Use a separate comparison structure instead.
- **Stack overflow on deep trees:** Very deep trees can exhaust the call stack in DFS. BFS is safer for unbalanced trees.
</Aside>

## Key insights for interviews

<Aside type="tip" title="Trade-offs to discuss">
- **DFS vs. BFS:** DFS is more elegant and concise; BFS is more iterative and safer for deep trees. Mention both when discussing the solution.
- **Null handling:** Symmetry requires checking both value equality and structure matching. A null on one side but not the other breaks symmetry.
- **Early termination:** Both solutions return false immediately upon detecting a mismatch, avoiding unnecessary traversals.
- **Space usage:** DFS uses call stack (height-bound); BFS uses explicit queue (width-bound). On a balanced tree, both are similar; on a skewed tree, BFS can be better.
- **Related problem:** Compare with #100 (Same Tree), which checks if two trees are identical, not mirrored. The key difference is the pairing logic.
</Aside>

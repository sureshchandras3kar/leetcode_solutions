## Problem Statement

Given an integer array `nums`, rotate the array to the right by `k` steps, where `k` is non-negative.

You must do this **in-place** with O(1) extra space if possible, although O(n) space is also acceptable.

<Aside type="tip" title="What to notice first">
- The rotation is **to the right**, meaning elements move forward and wrap around.
- "In-place" means modifying the original array without using extra arrays (except for the reverse approach which doesn't).
- When `k` is larger than the array length, only `k % n` steps matter due to circular nature.
- Example: rotating `[1,2,3,4,5]` right by 2 gives `[4,5,1,2,3]`.
</Aside>

### Examples

| Input | k | Output | Explanation |
| --- | --- | --- | --- |
| `[1,2,3,4,5]` | `2` | `[4,5,1,2,3]` | Rotate right 2 steps: last 2 elements move to front |
| `[99,100,1,2,3]` | `2` | `[2,3,99,100,1]` | Last 2 elements wrap to front |
| `[1]` | `0` | `[1]` | Single element, no rotation needed |
| `[1,2,3,4,5]` | `7` | `[4,5,1,2,3]` | k=7 % 5 = 2, same as rotating by 2 |

## Constraints

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^9`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Circular buffer implementation** — rotating elements in a fixed-size buffer used in streaming data processing.
- **Image rotation** — rotating pixel arrays for image transformations in computer graphics.
- **Playlist cycling** — rotating song lists in music players or media systems.
- **Ring buffer management** — managing write positions in circular data structures used in networking and real-time systems.
- **Time series windowing** — sliding windows of data in analytics where historical data wraps around.
</Aside>

## Approach 1: Reverse Trick (Recommended)

<ApproachTag tag="recommended" />

The key insight: reversing subarrays achieves rotation. For array `[1,2,3,4,5]` rotated right by 2:
1. Reverse entire array → `[5,4,3,2,1]`
2. Reverse first k elements → `[3,4,5,2,1]`
3. Reverse remaining n-k elements → `[3,4,5,1,2]`

This is pure in-place magic with O(1) extra space!

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass with reversals" spaceNote="No extra arrays" />

### Pseudocode

```plaintext
function rotateReverse(nums, k):
    if nums is empty or k == 0:
        return
    
    n = length of nums
    k = k % n  // Handle k > n
    if k == 0:
        return
    
    function reverse(start, end):
        while start < end:
            swap nums[start] and nums[end]
            start++
            end--
    
    reverse(0, n - 1)      // Reverse entire array
    reverse(0, k - 1)      // Reverse first k elements
    reverse(k, n - 1)      // Reverse remaining elements
```

## Approach 2: Extra Space

<ApproachTag tag="simple" />

Create a new array and place each element at its rotated position: element at index `i` goes to index `(i + k) % n`. Then copy back to the original array.

This approach is straightforward: compute where each element should go, place it there, and restore the original array reference.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Two passes" spaceNote="Extra array needed" />

### Step-by-step Example

**Input:** `nums = [1,2,3,4,5]`, `k = 2`

| i | Original Value | New Index (i+k)%n | rotated[newIndex] |
| --- | --- | --- | --- |
| 0 | 1 | (0+2)%5 = 2 | rotated[2] = 1 |
| 1 | 2 | (1+2)%5 = 3 | rotated[3] = 2 |
| 2 | 3 | (2+2)%5 = 4 | rotated[4] = 3 |
| 3 | 4 | (3+2)%5 = 0 | rotated[0] = 4 |
| 4 | 5 | (4+2)%5 = 1 | rotated[1] = 5 |

Result: `rotated = [4,5,1,2,3]` → copy back to `nums` ✓

### Pseudocode

```plaintext
function rotateExtraSpace(nums, k):
    if nums is empty or k == 0:
        return
    
    n = length of nums
    k = k % n  // Handle k > n
    if k == 0:
        return
    
    rotated = new array of size n
    for i in range(n):
        rotated[(i + k) % n] = nums[i]
    
    for i in range(n):
        nums[i] = rotated[i]
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting k % n**: When k > n, the rotation repeats. Only k % n steps actually change the array.
- **Off-by-one errors in reverse**: Ensure reverse boundaries are `[start, end]` inclusive. A common bug is using `end` instead of `end - 1`.
- **Confusing directions**: "Rotate right" means elements move toward higher indices (with wrapping). `[1,2,3]` rotated right by 1 is `[3,1,2]`, not `[2,3,1]`.
- **In-place mutations**: If not careful, the reverse trick's order matters. Reversing in a different order produces incorrect results.
- **Not modifying the original array**: In Python, lists are passed by reference, but returning a new list doesn't modify the original. The solution should modify in-place.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why reverse trick over extra space?** The reverse trick achieves true O(1) space (except for the recursive call stack in some languages). This is the optimal solution and interview gold.
- **Why is k % n important?** Rotating by n steps gets you back to the original, so only k % n matters. This edge case prevents unnecessary iterations.
- **Comparing to other rotation problems**: Problem #48 (Rotate Image) rotates a 2D matrix in-place; Problem #61 (Rotate List) does it on a linked list. The reverse trick generalizes across different data structures.
- **Edge cases to cover**: Empty arrays, k=0, k=n, k>n, arrays with single elements, and duplicate elements. All approaches handle duplicates naturally.
- **Follow-up — GCD approach**: There's a third approach using the GCD (Greatest Common Divisor) for cyclic rotation, which places each element directly in its final position. It's more complex but demonstrates deeper algorithmic thinking.
</Aside>

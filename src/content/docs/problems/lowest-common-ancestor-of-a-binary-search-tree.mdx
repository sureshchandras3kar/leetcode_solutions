## Problem Statement

Given a binary search tree (BST) of unique values and two nodes `p` and `q`, return the **lowest common ancestor** (LCA) of the two nodes in the BST.

The lowest common ancestor between two nodes p and q is the lowest node in a tree T such that both p and q are descendants of node (where we allow **a node to be a descendant of itself**).

<Aside type="tip" title="What to notice first">
- The BST property guarantees a **unique path** from root to any node.
- If both `p` and `q` are less than the current node, the LCA is in the **left subtree**.
- If both `p` and `q` are greater than the current node, the LCA is in the **right subtree**.
- Otherwise, the current node is the **LCA** (they diverge here).
</Aside>

### Examples

| Input | p | q | Output | Explanation |
| --- | --- | --- | --- | --- |
| Tree: `6, 2→8, 0, 4→7, 9, 3, 5` | 2 | 8 | 6 | 6 is the LCA of 2 and 8 |
| Tree: `6, 2→8, 0, 4→7, 9, 3, 5` | 2 | 4 | 2 | 2 is the LCA of 2 and 4 (one node is ancestor of the other) |
| Tree: `2, 1, 3` | 1 | 3 | 2 | 2 is the LCA of 1 and 3 |

## Constraints

- The number of nodes in the tree is in the range `[2, 10^5]`.
- `-10^9 <= Node.val <= 10^9`
- All Node.val are **unique**.
- `p != q`
- Both `p` and `q` exist in the BST.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **File system hierarchies** — find the common parent directory of two files to determine shared permissions.
- **Organizational structures** — identify the smallest management unit responsible for two employees.
- **Biological taxonomy** — find the most recent common ancestor of two species in an evolutionary tree.
- **Version control systems** — determine the merge base between two branches in Git history.
</Aside>

## Approach 1: Recursive (Recommended)

<ApproachTag tag="recommended" />

Leverage the BST property: compare current node value with both `p` and `q`. If both are smaller, recurse left. If both are larger, recurse right. Otherwise, the current node is the LCA.

The elegance comes from the fact that the BST structure guarantees a convergence point where the two nodes "split" — this point is the LCA.

<ComplexityBadges time="O(log n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Binary search-like" spaceNote="Recursion stack" />

### Step-by-step Example

**Input:** BST with root = 6, p = 2, q = 8

| Step | Current Node | p vs Node | q vs Node | Action |
| --- | --- | --- | --- | --- |
| 1 | 6 | 2 < 6 | 8 > 6 | p and q on different sides → return 6 |

**Result:** LCA = 6 ✓

**Input:** BST with root = 6, p = 2, q = 4

| Step | Current Node | p vs Node | q vs Node | Action |
| --- | --- | --- | --- | --- |
| 1 | 6 | 2 < 6 | 4 < 6 | Both in left subtree → recurse left |
| 2 | 2 | 2 == 2 | 4 > 2 | p and q on different sides → return 2 |

**Result:** LCA = 2 ✓

### Pseudocode

```plaintext
function lowestCommonAncestorRecursive(root, p, q):
    if root.val > p.val and root.val > q.val:
        // Both p and q are in left subtree
        return lowestCommonAncestorRecursive(root.left, p, q)
    elif root.val < p.val and root.val < q.val:
        // Both p and q are in right subtree
        return lowestCommonAncestorRecursive(root.right, p, q)
    else:
        // p and q are on different sides or one is root
        return root
```

## Approach 2: Iterative

<ApproachTag tag="interview" />

Same logic as the recursive approach, but implemented with a `while` loop instead of recursion. Navigate the tree by comparing node values with `p` and `q`, moving left or right until convergence.

This approach avoids the recursion stack entirely, useful when stack depth is a concern or when iterative solutions are preferred.

<ComplexityBadges time="O(log n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Binary search-like" spaceNote="No extra structure" />

### Step-by-step Example

Same as Approach 1, but using a while loop instead of recursion:

| Iteration | Current Node | p vs Node | q vs Node | Action |
| --- | --- | --- | --- | --- |
| 1 | 6 | 2 < 6 | 8 > 6 | p and q on different sides → return 6 |

### Pseudocode

```plaintext
function lowestCommonAncestorIterative(root, p, q):
    current = root
    while current is not null:
        if current.val > p.val and current.val > q.val:
            // Both in left subtree
            current = current.left
        elif current.val < p.val and current.val < q.val:
            // Both in right subtree
            current = current.right
        else:
            // Divergence point found
            return current
    return root
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting the BST property:** Treating it like a general binary tree and searching both subtrees. The BST allows pruning.
- **Comparing with both p and q separately:** Make sure you're comparing the **node value** against **both p.val and q.val**, not node structure.
- **Not handling the case where one node is the ancestor of another:** If p = 2 and q = 4, and 2 is an ancestor of 4, then 2 is the LCA, not their "convergence."
- **Assuming convergence always happens at an internal node:** The LCA can be one of the nodes themselves (p or q).
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why BST LCA is simpler than general tree LCA:** The BST property creates a unique path to each node. You don't need to search both subtrees; you can prune based on value comparisons.
- **Recursive vs Iterative:** Recursive is cleaner and more elegant. Iterative avoids stack overhead. Choose based on the interviewer's preference or constraints.
- **Edge cases:** One node is the ancestor of another (still valid LCA), both nodes are the same node is not possible (p != q guaranteed), null checks are minimal due to problem constraints.
- **Follow-up — General Binary Tree:** If the interviewer changes it to a non-BST, explain how you'd store parent pointers or use DFS to find ancestors, then find the first common one.
</Aside>

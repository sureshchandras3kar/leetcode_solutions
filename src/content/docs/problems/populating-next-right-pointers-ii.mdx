## Problem Statement

Given a binary tree (not necessarily perfect), populate each next pointer to point to its next right node. If there is no next right node, the next pointer should be set to NULL.

Initially, all next pointers are set to NULL.

<Aside type="tip" title="What to notice first">
- Unlike #116 (perfect binary tree), this is for any binary tree structure.
- The tree may have missing nodes, making alignment trickier.
- Both approaches use the property that a parent's level is already connected before processing children.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[1,2,3,4,5,null,7]` | Next pointers at level 2 connect `4→5` and `5→7` | Nodes at same level connected; null skipped |
| `[1,2,3]` | Simple tree: level 2 has `2→3` | Both nodes present, connection straightforward |

## Constraints

- The number of nodes is in the range `[0, 6000]`.
- `-100 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Tree serialization** — traversing levels for compact binary representation.
- **Breadth-first processing** — needing siblings at same level quickly.
- **Memory layout optimization** — cache-aware tree traversal linking related data.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Level-Order Queue | O(n) | O(w) | Simple, clear level traversal; w is max width |
| Pre-Computed Links | O(n) | O(1) | Memory-constrained; use parent-level links to traverse |

## Which approach should you learn first?

- Learn **Level-Order Queue** first — it is straightforward and intuitive.
- Study **Pre-Computed Links** to master O(1) space without extra queue.

## Approach 1: Level-Order Queue BFS

<ApproachTag tag="recommended" />

Use a queue to process nodes level-by-level. For each level, connect consecutive nodes' next pointers.

<ComplexityBadges time="O(n)" space="O(w)" timeColor="green" spaceColor="yellow" timeNote="Visit each node once" spaceNote="Queue stores max level width" />

### Step-by-step Example

**Input:** `[1,2,3,4,5,null,7]`

| Step | Action | Result |
| --- | --- | --- |
| Level 0 | Process node 1 | Node 1 next = NULL |
| Level 1 | Process nodes 2, 3 | Connect 2 → 3 |
| Level 2 | Process nodes 4, 5, 7 | Connect 4 → 5 → 7 (skip null) |

### Pseudocode

```plaintext
function connect(root):
    if not root: return root
    queue = [root]
    
    while queue not empty:
        levelSize = queue.length
        prev = null
        
        for i in 0..levelSize-1:
            node = queue.pop()
            if prev: prev.next = node
            prev = node
            
            if node.left: queue.push(node.left)
            if node.right: queue.push(node.right)
    
    return root
```

## Approach 2: Pre-Computed Links O(1) Space

<ApproachTag tag="optimal" />

Use next pointers from the parent level to traverse the current level, eliminating the queue.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Visit each node once" spaceNote="Only pointers, no extra storage" />

### Step-by-step Example

**Input:** `[1,2,3,4,5,null,7]`

| Step | Using | Connects |
| --- | --- | --- |
| Level 1 | root's structure | 2 → 3 |
| Level 2 | 2.next (= 3) | 4 → 5 → 7 |

### Pseudocode

```plaintext
function connect(root):
    if not root: return root
    leftmost = root
    
    while leftmost:
        prev = null
        current = leftmost
        
        while current:
            if current.left:
                if prev: prev.next = current.left
                prev = current.left
            
            if current.right:
                if prev: prev.next = current.right
                prev = current.right
            
            current = current.next
        
        leftmost = leftmost.left
    
    return root
```

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to handle null children — skip them but still link the next non-null child.
- Not resetting `prev` correctly at each level boundary.
- Using level-order queue when O(1) space solution exists.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why Level-Order Queue?** It is intuitive and handles irregular trees well.
- **Why Pre-Computed Links?** Demonstrates O(1) space mastery and efficient tree manipulation.
- **Edge cases:** Root is null, single node, tree with only left/right children.
- **Follow-up:** How would you handle a graph with arbitrary parent pointers?

## Problem Statement

A phrase is a palindrome if, after converting all uppercase letters into lowercase letters and removing all non-alphanumeric characters, it reads the same forward and backward. Alphanumeric characters include letters and numbers. Given a string `s`, return `true` if it is a palindrome, or `false` otherwise.

<Aside type="tip" title="What to notice first">
- "Alphanumeric" means letters (a-z, A-Z) and digits (0-9) — skip spaces, punctuation, etc.
- Always convert to the same case before comparing.
- Two pointers avoids creating a cleaned copy → O(1) extra space.
</Aside>

### Examples

| Input | Output | Reason |
| --- | --- | --- |
| `"A man, a plan, a canal: Panama"` | `true` | Cleaned: `"amanaplanacanalpanama"` is a palindrome |
| `"race a car"` | `false` | Cleaned: `"raceacar"` is not a palindrome |
| `" "` | `true` | After cleaning: `""` — empty string is a palindrome |

## Constraints

- `1 <= s.length <= 2 × 10^5`
- `s` consists only of printable ASCII characters

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Input sanitization** — checking if a DNA sequence or serial number is palindromic after stripping delimiters
- **Symmetric routing** — checking if a path identifier is the same forward and backward
- **Data integrity** — palindromic checksums in some encoding schemes
- **Natural language processing** — detecting palindromes in text after normalization
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Two Pointers | O(n) | O(1) | Preferred — no extra allocation |
| Clean Then Check | O(n) | O(n) | Clarity, simpler code |

## Approach 1: Two Pointers (Recommended)

<ApproachTag tag="recommended" />

Place left at the start and right at the end of the string. Skip non-alphanumeric characters on both sides, then compare the lowercased characters at each pointer. If any mismatch is found, return false. If the pointers meet or cross, return true.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceNote="No extra allocation" />

### Step-by-step Example

**Input:** `s = "A man, a plan, a canal: Panama"`

Cleaned view (not actually built): `"amanaplanacanalpanama"`

| Step | left char | right char | match? | action |
| --- | --- | --- | --- | --- |
| 1 | `'A'` → ``a`` | `'a'` | ✓ | advance both |
| 2 | ``m`` | `'m'` | ✓ | advance both |
| 3 | ``a`` | `'a'` | ✓ | advance both |
| ... | right pointer skips `':'`, `' '` | left skips `','`, `' '` | ... | skip non-alphanumeric |
| last | pointers meet | — | — | return `true` |

**Key behavior — skipping non-alphanumeric characters:**

When the left pointer lands on `','` or `' '`, the inner while loop advances it until it finds an alphanumeric character without doing any comparison. The same happens on the right side.

### Pseudocode

```plaintext
function isPalindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and not isAlphanumeric(s[left]):
            left++
        while left < right and not isAlphanumeric(s[right]):
            right--
        if lower(s[left]) != lower(s[right]):
            return false
        left++; right--
    return true
```

## Approach 2: Clean Then Check

<ApproachTag tag="simple" />

Filter the string to keep only alphanumeric characters, lowercase them all, then check if the resulting string equals its own reverse. Simple and readable, but allocates O(n) extra space.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Single pass" spaceNote="Cleaned string storage" />

### Pseudocode

```plaintext
function isPalindrome(s):
    cleaned = [c.lower() for c in s if c.isalnum()]
    return cleaned == reversed(cleaned)
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to handle digits — `'0'`–`'9'` ARE alphanumeric and must not be skipped.
- Not converting to lowercase before comparing (`'A' != 'a'`).
- Empty string edge case: an empty string after cleaning is a valid palindrome — return `true`.
</Aside>

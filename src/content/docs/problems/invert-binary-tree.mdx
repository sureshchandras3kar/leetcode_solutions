## Problem Statement

Given the `root` of a binary tree, invert the tree, and return its root.

Inverting a binary tree means swapping the left and right child nodes of every node in the tree.

<Aside type="tip" title="What to notice first">
- Every node in the tree must have its left and right children swapped.
- The tree structure remains the same, only the direction of children changes.
- This is a simple problem that teaches fundamental tree traversal techniques.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[4,2,7,1,3,6,9]` | `[4,7,2,9,6,3,1]` | All left-right children are swapped at each node |
| `[2,1,3]` | `[2,3,1]` | Left and right children of root are swapped |
| `[]` | `[]` | Empty tree remains empty |

## Constraints

- The number of nodes in the tree is in the range `[0, 100]`.
- `-100 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Mirror image processing** — create a mirror/flipped version of a hierarchical structure like organizational charts.
- **Game tree reflection** — flip the decision tree in game AI for symmetry analysis.
- **UI layout mirroring** — invert left/right layout for right-to-left (RTL) language support.
- **Data structure optimization** — balance tree traversal patterns for hardware cache efficiency.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Type | Best When |
| --- | --- | --- | --- | --- |
| DFS Recursive | O(n) | O(h) | Elegant | General case — simple and intuitive |
| BFS Iterative | O(n) | O(w) | Queue-based | Level-by-level processing preferred |

*h = height of tree, w = max width at any level*

## Which approach should you learn first?

- Pick **DFS Recursive** if you want the cleanest, most readable solution.
- Pick **BFS Iterative** if you prefer avoiding recursion or need level-order processing.

## Approach 1: DFS Recursive (Recommended)

<ApproachTag tag="recommended" />

Recursively visit each node, swap its left and right children, then continue the process for its subtrees.

The elegance of this approach lies in its simplicity: three lines of code (swap + two recursive calls) handle any tree size.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Visit each node once" spaceNote="Call stack depth" />

### Step-by-step Example

**Input:** `root = [4,2,7,1,3,6,9]`

```
Original tree:       4           Inverted tree:       4
                   /   \                            /   \
                  2     7                          7     2
                 / \   / \                        / \   / \
                1   3 6   9                      9   6 3   1
```

| Step | Action |
| --- | --- |
| 1 | Visit node 4: swap its children (2 ↔ 7) |
| 2 | Recurse on new left child (7): swap (6 ↔ 9) |
| 3 | Recurse on new right child (2): swap (1 ↔ 3) |
| 4 | Return to leaf nodes (no children to swap) |

### Pseudocode

```plaintext
function invertTreeDFS(root):
    if root is null:
        return null
    
    swap(root.left, root.right)
    invertTreeDFS(root.left)
    invertTreeDFS(root.right)
    
    return root
```

## Approach 2: BFS Iterative

<ApproachTag tag="interview" />

Use a queue to visit nodes level by level, swapping children at each node without using the recursion call stack.

This approach is useful when you want explicit control over traversal order or need to avoid stack overflow on very deep trees.

<ComplexityBadges time="O(n)" space="O(w)" timeColor="green" spaceColor="yellow" timeNote="Visit each node once" spaceNote="Queue width" />

### Step-by-step Example

**Input:** `root = [2,1,3]`

After processing:

| Step | Queue | Current Node | Action |
| --- | --- | --- | --- |
| 1 | `[2]` | 2 | Dequeue 2, swap (1 ↔ 3), enqueue [3, 1] |
| 2 | `[3, 1]` | 3 | Dequeue 3, no children, continue |
| 3 | `[1]` | 1 | Dequeue 1, no children, continue |
| 4 | `[]` | - | Queue empty, done |

### Pseudocode

```plaintext
function invertTreeBFS(root):
    if root is null:
        return root
    
    queue = Queue()
    queue.enqueue(root)
    
    while queue is not empty:
        node = queue.dequeue()
        swap(node.left, node.right)
        
        if node.left is not null:
            queue.enqueue(node.left)
        if node.right is not null:
            queue.enqueue(node.right)
    
    return root
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting to swap before recursing** — make sure the swap happens at the start, before processing children.
- **Only processing one subtree** — remember to recursively invert both left and right subtrees.
- **Returning the wrong node** — return the modified root, not one of its children.
- **In BFS:** forgetting to add children to the queue before dequeuing the next node.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why DFS over BFS?** Cleaner code, works well for most trees, and matches the tree's natural recursive structure. Use unless you have a specific reason to avoid recursion.
- **Why BFS?** Explicit control, avoids deep recursion (good for very deep trees), and enables level-order processing if needed.
- **Space complexity nuance:** For balanced trees, both are O(log n) due to tree shape. For skewed trees, DFS could approach O(n) with deep recursion, while BFS adapts to tree width.
- **Follow-up — Flatten to linked list:** A natural extension combining inversion with other tree transformations.
- **Follow-up — Spiral level order:** Combine BFS with direction reversal for zigzag traversal.
</Aside>

## Problem Statement

You are given an array of non-overlapping intervals `intervals` where `intervals[i] = [starti, endi]` represent the start and end of the ith interval. You are also given a new interval `newInterval = [newStart, newEnd]` that may overlap with some of the existing intervals.

**Insert** `newInterval` into `intervals` such that `intervals` is still sorted and non-overlapping (merge overlapping intervals as necessary). Return the resulting array of intervals.

<Aside type="tip" title="What to notice first">
- Three phases: (1) add non-overlapping intervals before newInterval, (2) merge overlapping intervals, (3) add remaining non-overlapping intervals.
- No need to sort; input is already sorted.
- Overlapping condition: `interval.end >= newInterval.start` AND `interval.start <= newInterval.end`.
</Aside>

### Examples

| intervals | newInterval | Output | Explanation |
| --- | --- | --- | --- |
| `[[1,2],[3,5],[6,9]]` | `[2,5]` | `[[1,5],[6,9]]` | [2,5] overlaps with [1,2] and [3,5], merge to [1,5] |
| `[[1,5]]` | `[0,0]` | `[[0,0],[1,5]]` | [0,0] doesn't overlap, add before |
| `[[1,5]]` | `[5,7]` | `[[1,7]]` | [5,7] overlaps at boundary, merge |
| `[[1,2],[3,5],[6,9]]` | `[10,11]` | `[[1,2],[3,5],[6,9],[10,11]]` | [10,11] doesn't overlap, append |

## Constraints

- `0 <= intervals.length <= 10^4`
- `intervals[i].length == 2`
- `0 <= starti <= endi <= 10^5`
- `newInterval.length == 2`
- `0 <= newStart <= newEnd <= 10^5`
- `intervals` is sorted by `start` and does not contain overlapping intervals

## Approach 1: Greedy Three-Phase (Recommended)

<ApproachTag tag="recommended" />

Divide the problem into three phases:
1. Add all intervals that end before `newInterval` starts.
2. Merge all overlapping intervals with `newInterval`.
3. Add remaining intervals that start after the merged interval ends.

This is efficient because the input is pre-sorted and non-overlapping.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Single scan through intervals" spaceNote="Result list storage" />

### Step-by-step Example

**Input:** `intervals = [[1,2],[3,5],[6,9]]`, `newInterval = [2,5]`

| Phase | Action | Result |
| --- | --- | --- |
| 1 | Add [1,2] (ends before 2) | [[1,2]] |
| 2 | Merge [2,5] with [3,5] → [2,5] | [[1,2], [2,5]] |
| 3 | Merge [2,5] with [6,9]? No (6 > 5) | [[1,2], [2,5]] |
| Final | Add [6,9] | [[1,2], [2,5], [6,9]] |

### Pseudocode

```plaintext
function insert(intervals, newInterval):
    result = []
    i = 0
    newStart, newEnd = newInterval
    
    // Phase 1: Add non-overlapping intervals before
    while i < len(intervals) and intervals[i][1] < newStart:
        result.append(intervals[i])
        i += 1
    
    // Phase 2: Merge overlapping intervals
    while i < len(intervals) and intervals[i][0] <= newEnd:
        newStart = min(newStart, intervals[i][0])
        newEnd = max(newEnd, intervals[i][1])
        i += 1
    result.append([newStart, newEnd])
    
    // Phase 3: Add remaining intervals
    while i < len(intervals):
        result.append(intervals[i])
        i += 1
    
    return result
```

<ApproachTag tag="simple" />

```plaintext
function insert(intervals, newInterval):

    i = 0
    newStart, newEnd = newInterval
    
    // Phase 1: Add intervals ending before new interval starts

        result.append(intervals[i])

    // Phase 2: Merge overlapping intervals

        i += 1
    result.append([newStart, newEnd])
    
    // Phase 3: Append remaining non-overlapping intervals

        result.append(intervals[i])
        i += 1
    
    return result
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Sorting the result:** The input is already sorted, so never sort again—it wastes O(n log n).
- **Off-by-one errors in phase boundaries:** Ensure `intervals[i][1] < newStart` (strictly less) and `intervals[i][0] <= newEnd` (less or equal).
- **Not updating newStart during merge:** Always take the minimum start; it expands the merged interval.
- **Forgetting to check phase 1 condition:** If phase 1 loop is skipped, newInterval should still be added.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why three phases?** Breaking into phases makes logic clear and avoids re-sorting.
- **Can you do this without modifying the list?** Yes, but you'd still need O(n) space for the result.
- **What if intervals overlapped in the input?** Merge them first (LeetCode 56), then insert.
- **What if newInterval is already in the list?** The algorithm handles it naturally by merging.
</Aside>

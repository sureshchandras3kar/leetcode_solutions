## Problem Statement

Given the `head` of a sorted linked list, **delete all nodes that have duplicate numbers**, leaving only **distinct numbers**. Return the head of the modified linked list.

<Aside type="tip" title="What to notice first">
- The list is **sorted**, which means all duplicate values are consecutive.
- You must **delete entire groups** of duplicates, not just reduce them to one — if a value appears 2+ times, it's gone entirely.
- A dummy node is crucial: if the head itself is part of a duplicate group, the dummy becomes the new head.
- Single-pass approach tracks "prev" (last non-duplicate node) and checks if the current and next nodes match.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `1 → 2 → 3 → 3 → 4 → 4 → 5` | `1 → 2 → 5` | Remove all `3`s and `4`s (appeared 2+ times) |
| `1 → 1 → 1 → 2 → 3` | `2 → 3` | Remove all `1`s |
| `1 → 1` | `(empty)` | Both nodes are duplicates; return empty list |

## Constraints

- The number of nodes in the list is in the range `[0, 300]`.
- `-100 <= Node.val <= 100`
- The list is guaranteed to be sorted in ascending order.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Log deduplication** — remove entries that appear multiple times in a sorted log file.
- **Database cleanup** — strip out duplicate transaction records that violate uniqueness constraints.
- **Version control** — remove commits that share the same message hash from a history.
- **Signal processing** — filter out duplicate measurements in a time-series stream.
</Aside>

## Approach 1: Single Pass (Two Pointers)

<ApproachTag tag="recommended" />

Use a dummy node and a `prev` pointer. Iterate through the list; when you find a duplicate (current and next have the same value), skip all nodes with that value. Otherwise, advance `prev` and move to the next node.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="One pass through list" spaceNote="Only pointer variables" />

### Step-by-step Example

**Input:** `1 → 2 → 3 → 3 → 4 → 4 → 5`

| Step | Prev | Current | Action | Note |
| --- | --- | --- | --- | --- |
| 1 | dummy | 1 | 1 ≠ 2 → advance prev | 1 is unique |
| 2 | 1 | 2 | 2 ≠ 3 → advance prev | 2 is unique |
| 3 | 2 | 3 | 3 = 3 → skip all 3s | Duplicate detected |
| 4 | 2 | 4 | 4 = 4 → skip all 4s | Duplicate detected |
| 5 | 2 | 5 | No next → done | 5 is unique |
| **Result** | | | dummy → 1 → 2 → 5 | Duplicates removed |

### Pseudocode

```plaintext
function deleteDuplicates(head):
    dummy = new ListNode(0)
    dummy.next = head
    prev = dummy
    current = head
    
    while current != null:
        if current.next != null and current.val == current.next.val:
            // Skip all duplicates
            value = current.val
            while current != null and current.val == value:
                current = current.next
            // Link prev to the first non-duplicate node
            prev.next = current
        else:
            // Unique node, keep it
            prev = prev.next
            current = current.next
    
    return dummy.next
```

## Approach 2: Hash Set

<ApproachTag tag="simple" />

First pass: count the frequency of each value using a set or hash map. Second pass: build the result list, skipping any node whose value appeared more than once.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Two passes" spaceNote="Hash set stores unique values" />

### Step-by-step Example

**Input:** `1 → 2 → 3 → 3 → 4 → 4 → 5`

| Step | Value | Frequency | Keep? |
| --- | --- | --- | --- |
| Pass 1 | 1 | 1 | Yes |
| | 2 | 1 | Yes |
| | 3 | 2 | No |
| | 4 | 2 | No |
| | 5 | 1 | Yes |
| **Pass 2** | Keep: `1 → 2 → 5` | | |

### Pseudocode

```plaintext
function deleteDuplicates(head):
    // Count frequencies
    freq = {}
    current = head
    while current != null:
        freq[current.val] = freq.get(current.val, 0) + 1
        current = current.next
    
    // Build result list
    dummy = new ListNode(0)
    dummy.next = head
    prev = dummy
    current = head
    
    while current != null:
        if freq[current.val] == 1:
            prev = current
            current = current.next
        else:
            // Skip this node (duplicate)
            prev.next = current.next
            current = current.next
    
    return dummy.next
```

<Aside type="caution" title="Watch for these pitfalls">
- **Dummy node is essential:** If the head itself is part of a duplicate group, the dummy becomes the safe starting point.
- **Advancing current vs prev:** In single-pass, when you skip duplicates, only current moves; prev stays until a unique node is found.
- **Null pointer checks:** Always verify `current.next != null` before comparing values to avoid crashes.
- **Forgetting the second skip:** When skipping duplicates, you must advance current *through all matching values*, not just to the next node.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Single-pass benefit:** O(1) space is ideal when memory is constrained.
- **Hash set benefit:** Two separate passes make the logic clearer and easier to debug.
- **Dummy node insight:** Using a sentinel node simplifies edge cases (e.g., duplicate at head).
- **When to use which:** Ask the interviewer about space constraints; if O(n) space is acceptable, the hash set approach is easier to code error-free.
- **Edge case:** A list of all duplicates (e.g., `1 → 1 → 1`) should return an empty list—verify this works in your implementation.

## Problem Statement

You are given an integer array `coins` representing coins of different denominations and an integer `amount` representing a total amount of money.

Return the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return `-1`.

You may assume that you have an infinite number of each kind of coin.

<Aside type="tip" title="What to notice first">
- This is an optimization problem: minimize the number of coins.
- You can use each coin denomination unlimited times.
- If the amount cannot be made, return -1.
- This is a classic DP problem that can also be solved with BFS (shortest path).
</Aside>

### Examples

| coins | amount | Output | Explanation |
| --- | --- | --- | --- |
| `[1,2,5]` | `5` | `1` | One 5-coin makes 5 |
| `[2]` | `3` | `-1` | Cannot make 3 with only 2-coins |
| `[10]` | `10` | `1` | One 10-coin makes 10 |
| `[1,3,4]` | `6` | `2` | Two 3-coins or one 3-coin and three 1-coins |

## Constraints

- `1 <= coins.length <= 12`
- `1 <= coins[i] <= 2^31 - 1`
- `0 <= amount <= 10^4`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Currency exchange** — minimizing number of bills/coins to make a payment.
- **Resource allocation** — using minimum resources to reach a target capacity.
- **Scheduling** — minimizing number of time slots to cover a duration.
- **Inventory management** — minimum packaging units needed for a quantity.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best For |
| --- | --- | --- | --- |
| DP | O(n * amount) | O(amount) | Classic approach, intuitive logic |
| BFS | O(n * amount) | O(amount) | Shortest path perspective, level-based traversal |

where n = len(coins)

## Approach 1: Dynamic Programming

<ApproachTag tag="recommended" />

dp[i] = minimum coins needed to make amount i.

For each amount i, try all coins and pick the one that results in the minimum number of coins.

<ComplexityBadges time="O(n * amount)" space="O(amount)" timeColor="yellow" spaceColor="yellow" timeNote="Double loop" spaceNote="DP array" />

### Step-by-step Example

**Input:** `coins = [1, 3, 4]`, `amount = 6`

| amount | coin=1 | coin=3 | coin=4 | dp[amount] |
| --- | --- | --- | --- | --- |
| 0 | — | — | — | 0 |
| 1 | 1+dp[0]=1 | N/A | N/A | 1 |
| 2 | 1+dp[1]=2 | N/A | N/A | 2 |
| 3 | 1+dp[2]=3 | 1+dp[0]=1 | N/A | 1 |
| 4 | 1+dp[3]=2 | 1+dp[1]=2 | 1+dp[0]=1 | 1 |
| 5 | 1+dp[4]=2 | 1+dp[2]=3 | 1+dp[1]=2 | 2 |
| 6 | 1+dp[5]=3 | 1+dp[3]=2 | 1+dp[2]=3 | 2 |

**Result:** dp[6] = 2 (two 3-coins)

### Pseudocode

```plaintext
function coinChangeDP(coins, amount):
    dp = array of size amount + 1, all initialized to infinity
    dp[0] = 0
    
    for i from 1 to amount:
        for coin in coins:
            if coin <= i and dp[i - coin] is not infinity:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] is not infinity else -1
```

## Approach 2: Breadth-First Search

<ApproachTag tag="interview" />

Treat this as a shortest path problem: start at `amount` and work backward, trying each coin denomination. The first time we reach 0, we've found the minimum coins.

BFS explores level by level, where level k contains all amounts reachable with k coins.

<ComplexityBadges time="O(n * amount)" space="O(amount)" timeColor="yellow" spaceColor="yellow" timeNote="Queue and visited set" spaceNote="Visited set" />

### Key Insight

BFS guarantees finding the shortest path (minimum coins) because it explores all possibilities at distance k before exploring at distance k+1.

### Pseudocode

```plaintext
function coinChangeBFS(coins, amount):
    if amount == 0:
        return 0
    
    queue = [(amount, 0)]  # (remaining_amount, num_coins)
    visited = {amount}
    
    while queue is not empty:
        curr_amount, num_coins = queue.pop_front()
        
        for coin in coins:
            next_amount = curr_amount - coin
            if next_amount == 0:
                return num_coins + 1
            if next_amount > 0 and next_amount not in visited:
                visited.add(next_amount)
                queue.push_back((next_amount, num_coins + 1))
    
    return -1
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Initializing dp values to 0 instead of infinity, leading to incorrect minimums.
- Forgetting to initialize dp[0] = 0 (0 coins needed to make 0 amount).
- Not checking if dp[i - coin] is valid before using it in comparisons.
- In BFS, forgetting to mark visited amounts to avoid revisiting and infinite loops.
- Returning 0 instead of -1 when amount cannot be made.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **DP vs BFS:** Both have the same complexity, but DP is iterative (bottom-up) and BFS is level-based. DP is typically preferred in interviews.
- **Space optimization:** DP uses O(amount) space. Some optimizations are possible if you only track amounts reachable with k coins, but it complicates the code.
- **Reconstruction:** To find the actual coins used, track the coin that resulted in each dp[i] value, then backtrack from dp[amount].
- **Edge cases:** amount = 0 (return 0), amount < 0 (return -1), no valid coins (return -1).
</Aside>

## Problem Statement

Given an integer array `nums` where the elements are **sorted in ascending order**, convert it to a **height-balanced** binary search tree.

A height-balanced binary tree is a binary tree in which the depth of the two subtrees of every node never differs by more than one.

<Aside type="tip" title="What to notice first">
- Divide and conquer: Pick the middle element as the root to balance the tree.
- Recursively build left and right subtrees from the left and right halves.
- The sorted order guarantees BST property without additional checking.
- Any array element can be the root; the middle pick ensures balance.
</Aside>

### Examples

| Input | Output |
| --- | --- |
| `nums = [-10,-3,0,5,9]` | A balanced BST with root 0, left child -3, right child 5 |
| `nums = [1,3]` | A balanced BST with root 3, left child 1 |

## Constraints

- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` is sorted in a strictly increasing order.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Database indexing** — balanced BSTs are used for efficient lookups and range queries.
- **Memory allocation** — divide and conquer builds balanced data structures.
- **Game AI** — balanced trees for decision trees and minimax algorithms.
- **Networking** — balanced routing trees for efficient packet forwarding.
- **Configuration systems** — hierarchical data structures for settings management.
</Aside>

## Approach 1: Recursive Divide and Conquer (Recommended)

<ApproachTag tag="recommended" />

1. Find the middle element of the current range → becomes the root.
2. Recursively build the left subtree from the left half.
3. Recursively build the right subtree from the right half.
4. Base case: when left > right, return null.

<ComplexityBadges time="O(n)" space="O(log n)" timeColor="green" spaceColor="green" timeNote="Visit each node once" spaceNote="Recursion depth" />

### Pseudocode

```plaintext
function sortedArrayToBST(nums):
    return build(0, len(nums) - 1)

function build(left, right):
    if left > right:
        return null
    
    mid = (left + right) / 2
    root = TreeNode(nums[mid])
    root.left = build(left, mid - 1)
    root.right = build(mid + 1, right)
    return root
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Using integer division truncation instead of proper midpoint calculation.
- Forgetting the base case (left > right returns null).
- Building an unbalanced tree by always picking left or right instead of middle.
- Off-by-one errors in left/right pointer updates.
- Creating new TreeNode without assigning a value (then assigning later in iterative).
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Middle pick:** Guarantees height balance; any element works as root if BST property is maintained via sorted input.
- **Recursion depth:** Log(n) for balanced tree, worst case O(n) for skewed trees (avoided here by middle pick).
- **Follow-up:** How would you build from an unsorted array? (Sort first, then apply same logic.)
- **Space optimization:** Can you do it in-place? (Not typical for tree problems; still need O(log n) for recursion.)
- **Edge cases:** Single element array, two elements, negative numbers, large numbers.
</Aside>

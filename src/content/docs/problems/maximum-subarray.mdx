## Problem Statement

Given an integer array `nums`, find the subarray with the largest sum and return its sum. A subarray is a contiguous non-empty sequence of elements within an array.

<Aside type="tip" title="What to notice first">
- A subarray must be **contiguous** — you cannot skip elements.
- You are looking for the **largest sum**, not the largest array.
- Even if all numbers are negative, you must return the one with the smallest absolute value (least negative).
- The answer is never "no subarray" — you must return at least one element.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[-2, 1, -3, 4, -1, 2, 1, -5, 4]` | `6` | Subarray `[4, -1, 2, 1]` has the largest sum of 6 |
| `[5, 4, -1, 7, 8]` | `23` | Entire array `[5, 4, -1, 7, 8]` has the sum of 23 |
| `[-1]` | `-1` | Only one element, so return that element |

## Constraints

- `1 <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Stock trading** — finding the time period with the best cumulative gains.
- **Network traffic analysis** — identifying busiest periods by summing request counts over time windows.
- **Weather analysis** — finding consecutive days with the highest average temperature change.
- **Budget forecasting** — identifying quarters with the highest cumulative cash flow.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Learning, tiny arrays |
| Kadane | O(n) | O(1) | General case — fast and simple |
| Divide & Conquer | O(n log n) | O(log n) | Interview discussion |

## Which approach should you learn first?

- Pick **Kadane** for the optimal interview solution — it is simple, O(n), and elegant.
- Pick **Brute Force** if you are learning how to think about subarrays.
- Pick **Divide & Conquer** if you want to understand the recursive pattern or are discussing trade-offs.

## Approach 1: Brute Force

<ApproachTag tag="simple" />

Check every possible subarray by examining all pairs of start and end indices. For each starting position, consider all possible ending positions and track the maximum sum found.

<ComplexityBadges time="O(n²)" space="O(1)" timeColor="red" spaceColor="green" timeNote="All pairs checked" spaceNote="Constant" />

## Approach 2: Kadane's Algorithm (Recommended)

<ApproachTag tag="recommended" />

Use dynamic programming to track the maximum sum ending at the current position. If adding the current element to the previous maximum sum is worse than starting fresh, restart from the current element.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceNote="Constant" />

## Approach 3: Divide and Conquer

<ApproachTag tag="interview" />

Divide the array into two halves and recursively find the maximum subarray in each half. The answer is either in the left half, right half, or crosses the midpoint.

<ComplexityBadges time="O(n log n)" space="O(log n)" timeColor="yellow" spaceColor="yellow" timeNote="Recursive depth" spaceNote="Call stack" />

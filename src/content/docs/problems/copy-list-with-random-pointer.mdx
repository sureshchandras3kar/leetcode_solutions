## Problem Statement

A linked list of length `n` is given such that each node contains an additional random pointer, which could point to any node in the list, or null.

Construct a **deep copy** of the list. The deep copy should consist of exactly `n` **brand new** nodes, where each new node has its value set to the value of its corresponding original node. Both the `next` and `random` pointer of the new nodes should point to new nodes in the copy list such that the pointers in the copy list represent the same relationships as the pointers in the original list (based on the nodes in the original list, not on their values).

Return the head of the copied linked list.

<Aside type="tip" title="What to notice first">
- The difficulty is that you cannot simply traverse and copy — you must also correctly map random pointers to the cloned nodes, not the original ones.
- Nodes can appear in the random pointer chain in any order, so a naive single-pass approach will not work.
- The hash map approach explicitly maps each original node to its clone; the interweaving approach reuses the original list structure temporarily as a mapping.
- Two pointers in different lists are "equivalent" if they point to nodes with the same position in their respective lists.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[[7,null],[13,0],[11,4],[10,2],[1,0]]` | Same structure | 5-node list; node 13's random points to node 7 (index 0) in the original, and the copy's node 13's random points to the copy's node 7. |
| `[[1,1],[2,1]]` | Same structure | 2-node list; both nodes' random pointers point to node 1 in both original and copy. |
| `[[3,null]]` | Same structure | Single node with null random pointer. |

## Constraints

- `0 <= n <= 100`
- `-100 <= Node.val <= 100`
- `Node.random` is null or points to some node in the list.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Deep cloning of complex objects** — copy an object graph where references can point anywhere, not just linearly.
- **Serialization/deserialization** — reconstruct a full copy of a graph structure from encoded form.
- **Cache replication** — duplicate a cached structure with internal cross-references for distributed systems.
- **Transaction rollback snapshots** — create a snapshot of a linked data structure for recovery purposes.
</Aside>

## Approach 1: Hash Map

<ApproachTag tag="recommended" />

Use a hash map to store the mapping between original nodes and their clones. In the first pass, create all cloned nodes and store them in the map. In the second pass, connect the next and random pointers of the cloned nodes by looking up the original pointers in the map.

The key insight is to separate node creation from pointer assignment, allowing us to reference any node's clone regardless of traversal order.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Two passes" spaceNote="Hash map stores n nodes" />

### Step-by-step Example

**Input:** `[[7,null],[13,0],[11,4],[10,2],[1,0]]`

Original list: 7 → 13 → 11 → 10 → 1
- Node 7's random: null
- Node 13's random: 7 (index 0)
- Node 11's random: 1 (index 4)
- Node 10's random: 11 (index 2)
- Node 1's random: 7 (index 0)

| Step | Node | Action |
| --- | --- | --- |
| 1 | Create clones | Create nodes: 7', 13', 11', 10', 1' in hash map |
| 2 | Node 7 → 13 | Copy 7.next=13 → 7'.next=13' |
| 3 | Node 7 → null | Copy 7.random=null → 7'.random=null |
| 4 | Node 13 → 0 | Copy 13.random=nodes[0] → 13'.random=nodeMap[nodes[0]]=7' |

### Pseudocode

```plaintext
function copyRandomList(head):
    if head is null:
        return null
    
    nodeMap = {}
    
    // First pass: create all nodes
    current = head
    while current != null:
        nodeMap[current] = Node(current.val)
        current = current.next
    
    // Second pass: set pointers
    current = head
    while current != null:
        copy = nodeMap[current]
        copy.next = nodeMap[current.next] if current.next else null
        copy.random = nodeMap[current.random] if current.random else null
        current = current.next
    
    return nodeMap[head]
```

## Approach 2: Interweaving (Constant Space)

<ApproachTag tag="advanced" />

Instead of a hash map, interweave the clones into the original list temporarily. After the first pass, the structure is: `original → clone → original → clone → ...`. In the second pass, the original node's random neighbor is exactly one step away from the clone's random neighbor. In the third pass, separate the two lists and restore the original.

This technique eliminates external storage for the mapping by reusing the list structure itself as a pointer table.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Three passes" spaceNote="Only pointers, no extra map" />

### Step-by-step Example

**Input:** `[[7,null],[13,0]]`

Original: 7 → 13

| Step | Structure | Action |
| --- | --- | --- |
| 1 | 7 → 7' → 13 → 13' | Interleave clones |
| 2 | 7' → null, 13' → 7' | Set random pointers |
| 3 | 7 → 13 (restored), 7' → 13' → null | Separate lists |

### Pseudocode

```plaintext
function copyRandomListInterweave(head):
    if head is null:
        return null
    
    // Pass 1: Interweave
    current = head
    while current != null:
        copy = Node(current.val)
        copy.next = current.next
        current.next = copy
        current = copy.next
    
    // Pass 2: Set random pointers
    current = head
    while current != null:
        if current.random != null:
            current.next.random = current.random.next
        current = current.next.next
    
    // Pass 3: Separate lists
    current = head
    copyHead = head.next
    while current != null:
        copy = current.next
        current.next = copy.next
        copy.next = copy.next.next if copy.next else null
        current = current.next
    
    return copyHead
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting that random pointers can point backward, forward, or to themselves — they are not bound by list order.
- In the hash map approach, checking if a pointer exists before accessing the map; always use a map that safely returns null for missing keys.
- In the interweaving approach, losing track of the original list structure during separation; restore it carefully in pass 3.
- Confusing shallow copy (copying pointers) with deep copy (creating new nodes); this problem requires a full deep copy.
- Not handling the null head case or null pointers within the random field.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Hash Map vs. Interweaving:** Hash map is clearer and safer; interweaving is more elegant and space-optimal. Most interviewers accept either.
- **Why two passes minimum?** Random pointers can point forward or backward; you cannot build the copy in a single pass without either a map or temporary storage.
- **Handling cycles:** The algorithm does not assume the list is acyclic, so it correctly handles cases where random pointers create cycles (e.g., a node's random points back to itself).
- **Testing:** Always test with null head, single node, and random pointers that form cycles or span the entire list.
- **Distinguishing deep from shallow:** Explicitly mention that you are creating new nodes, not just copying pointers. Shallow copy would fail this problem.

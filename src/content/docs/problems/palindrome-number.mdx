## Problem Statement

Given an integer `x`, return `true` if `x` is a **palindrome**, and `false` otherwise.

A palindrome reads the same forwards and backwards.

<Aside type="tip" title="What to notice first">
- Negative numbers are never palindromes because of the `-` sign.
- A number like `10` is not a palindrome because reversing it gives `01`.
- The follow-up most interviewers care about is whether you can solve it without converting to a string.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `121` | `true` | Reads as 121 from left to right and right to left |
| `-121` | `false` | Reads as -121 forward but 121- backward |
| `10` | `false` | Reads as 01 from right to left |

## Constraints

- `-2^31 <= x <= 2^31 - 1`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- Database indexing and query optimization
- Buffer management in system memory
- Time-series data analysis and sliding windows
- Image pixel manipulation and batch processing
</Aside>

## Complexity Comparison

| Approach | Time | Space | Notes |
| --- | --- | --- | --- |
| Without String (math) | O(log n) | O(1) | n = number of digits; no string allocation |
| With String | O(n) | O(n) | Converts integer to string; straightforward but uses extra memory |

## Which approach should you learn first?

- Pick **Without String** if you want the strongest interview answer.
- Pick **With String** if you want the easiest version to understand before optimizing.

## Approach 1: Without String (Recommended)

<ApproachTag tag="recommended" />

Reverse the entire integer mathematically and compare it to the original. Early-exit for negative numbers and numbers ending in zero (except 0 itself).

Think of this as rebuilding the number from right to left. If the rebuilt number matches the original one, the digits were symmetrical.

<ComplexityBadges time="O(log n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="One pass per digit" spaceNote="Integer vars only" />

`121 == 121` → `true` ✓

**Input:** `x = -121`

Negative → return `false` immediately ✓

**Input:** `x = 10`

`10 % 10 == 0` (ends in zero, non-zero) → return `false` immediately ✓

### Pseudocode

```plaintext
function isPalindrome(x):
    if x < 0:
        return false
    if x == 0:
        return true
    if x % 10 == 0:
        return false
    original = x
    reversed = 0
    while x > 0:
        lastDigit = x % 10
        reversed = reversed * 10 + lastDigit
        x = x / 10
    return original == reversed
```

## Approach 2: With String

<ApproachTag tag="simple" />

Convert the integer to a string and compare it to its reverse.

This version is easy to read because it turns the problem into a string comparison. The trade-off is extra memory.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Per digit" spaceNote="String storage" />

### Step-by-step Example

**Input:** `x = 121`

`str(121)` → `"121"` → reversed → `"121"` → equal → `true` ✓

**Input:** `x = -121`

`str(-121)` → `"-121"` → reversed → `"121-"` → not equal → `false` ✓

**Input:** `x = 10`

`str(10)` → `"10"` → reversed → `"01"` → not equal → `false` ✓

### Pseudocode

```plaintext
function isPalindrome(x):
    str_x = str(x)
    return str_x == reverse(str_x)
```

## Interview Tips

<Aside type="tip" title="Key points to discuss">
- **Why avoid strings?** The problem statement hints at it: "could you solve it without converting the integer to a string?" The math approach is O(1) space vs O(n) for the string version.
- **Why the early exit for `x % 10 == 0`?** If a non-zero number ends in 0, reversing it would place a 0 at the front — impossible for a valid positive integer palindrome.
- **Overflow concern:** Reversing a large int can overflow a 32-bit integer. The C++ solution uses `long long` for the reversed value to handle this safely.
- **Negative numbers:** Always false — the `-` sign breaks palindrome symmetry.
</Aside>

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Treating negative numbers as valid palindromes.
- Forgetting the special case where a non-zero number ends in `0`.
- Describing the string solution as O(1) space when it creates a new string.

## Problem Statement

Given an integer array `nums`, return an array `answer` such that `answer[i]` is equal to the product of all the elements of `nums` except `nums[i]`. The product of any prefix or suffix of `nums` is guaranteed to fit in a 32-bit integer. You must write an algorithm that runs in O(n) time and without using the division operation.

<Aside type="tip" title="Key insight">
- Division would be the obvious answer but it is explicitly forbidden — and breaks when the array contains zeros.
- The key insight: `output[i] = (product of everything LEFT of i) × (product of everything RIGHT of i)`.
- You can compute both sides in separate passes without ever dividing.
</Aside>

### Examples

| Input | Output |
| --- | --- |
| `[1,2,3,4]` | `[24,12,8,6]` |
| `[-1,1,0,-3,3]` | `[0,-9,0,9,0]` |

## Constraints

- `2 <= nums.length <= 10^5`
- `-30 <= nums[i] <= 30`
- The product of any prefix or suffix fits in a 32-bit integer

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Parallel computation** — computing per-node contributions in distributed matrix operations.
- **Financial returns** — computing total portfolio return excluding each individual asset's contribution.
- **Leave-one-out cross validation** — machine learning metric where you exclude each data point's own contribution.
- **Signal processing** — computing running products that exclude self-interference.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Prefix/Suffix Arrays | O(n) | O(n) | Clarity matters, extra memory allowed |

## Approach 1: Prefix/Suffix Arrays

<ApproachTag tag="recommended" />

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Two passes" spaceNote="Two extra arrays" />

Build a `prefix` array where `prefix[i]` is the product of all elements to the left of index `i`, and a `suffix` array where `suffix[i]` is the product of all elements to the right of index `i`. The answer at each index is simply `prefix[i] * suffix[i]`.

### Pseudocode

```plaintext
function productExceptSelf(nums):
    n = len(nums)
    prefix = [1] * n
    suffix = [1] * n
    for i in range(1, n):
        prefix[i] = prefix[i-1] * nums[i-1]
    for i in range(n-2, -1, -1):
        suffix[i] = suffix[i+1] * nums[i+1]
    return [prefix[i] * suffix[i] for i in range(n)]
```

## Approach 2: Two-Pass O(1) Space

<ApproachTag tag="space-optimal" />

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" spaceNote="Output array only" />

Reuse the output array itself instead of allocating separate prefix and suffix arrays. The first pass (left→right) fills each slot with the running prefix product. The second pass (right→left) multiplies each slot by a running suffix product variable.

### Step-by-step Example

**Input:** `[1, 2, 3, 4]`

After pass 1 (left prefix into output): `output = [1, 1, 2, 6]`

Pass 2 (right→left, running suffix starts at 1):

| i | output[i] before | suffix | output[i] after | suffix update |
| --- | --- | --- | --- | --- |
| 3 | 6 | 1 | 6×1 = **6** | 1×4 = 4 |
| 2 | 2 | 4 | 2×4 = **8** | 4×3 = 12 |
| 1 | 1 | 12 | 1×12 = **12** | 12×2 = 24 |
| 0 | 1 | 24 | 1×24 = **24** | — |

Result: `[24, 12, 8, 6]`

### Pseudocode

```plaintext
function productExceptSelfOptimal(nums):
    n = len(nums)
    output = [1] * n
    prefix = 1
    for i in range(n):
        output[i] = prefix
        prefix *= nums[i]
    suffix = 1
    for i in range(n-1, -1, -1):
        output[i] *= suffix
        suffix *= nums[i]
    return output
```

<Aside type="caution" title="Common mistakes">
- Forgetting that `prefix[0]` and `suffix[n-1]` must both start as `1` (the multiplicative identity).
- Off-by-one: `prefix[i]` should be the product of `nums[0..i-1]`, not including `nums[i]`.
- Division approach: fails when the array contains zeros (division by zero).
</Aside>

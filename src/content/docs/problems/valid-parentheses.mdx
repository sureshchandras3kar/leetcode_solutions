## Problem Statement

Given a string `s` containing just the characters `'('`, `')'`, `'['`, `']'`, `'{'` and `'}'`, determine if the input string is valid.

An input string is valid if:
1. Open brackets must be closed by the same type of closing bracket.
2. Open brackets must be closed in the correct order.
3. Every close bracket has a corresponding open bracket of the same type.

<Aside type="tip" title="What to notice first">
- Each closing bracket must match the **most recent unmatched opening bracket** of the same type.
- This ordering requirement is the key insight — it's a perfect use case for a **stack** data structure.
- The stack approach is faster and more intuitive than alternatives.
</Aside>

### Examples

| Input | Valid? | Explanation |
| --- | --- | --- |
| `"()"` | true | Simple pair of parentheses |
| `"()[]{}"` | true | Three different pairs, all properly closed |
| `"(]"` | false | Closing bracket does not match opening bracket |
| `"([{}])"` | true | Nested brackets in correct order |
| `"([)]"` | false | Brackets closed in wrong order (interleaved) |
| `""` | true | Empty string is valid |
| `"("` | false | Unclosed opening bracket |
| `")"` | false | Closing bracket with no opening |

## Constraints

- `0 <= s.length <= 10^4`
- `s` consists of parentheses only: `'(', ')', '[', ']', '{', '}'`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Code editors and IDEs** — auto-closing brackets, syntax highlighting, bracket matching
- **Compilers and interpreters** — validating syntax trees and nested structures
- **HTML/XML parsing** — checking that tags are properly opened and closed
- **Mathematical expression evaluation** — ensuring parentheses are balanced before parsing
- **Text editors** — validating nested delimiters in code and markup
</Aside>

## Complexity Comparison

| Approach | Time | Space | Intuition | Best When |
| --- | --- | --- | --- | --- |
| Stack | O(n) | O(n) | Process left-to-right, match using memory | General case — fast and clear |
| Replace Loop | O(n²) | O(n) | Remove pairs repeatedly | Learning the concept, simple implementation |

- Pick **Stack** if you want the best all-around solution.
- Pick **Replace Loop** if you want to understand the concept through a brute-force lens.

## Approach 1: Stack (Recommended)

<ApproachTag tag="recommended" />

Use a stack to track opening brackets. When you encounter a closing bracket, check if it matches the most recent opening bracket on the stack.

**Key insight:** The last opened bracket that hasn't been closed yet must match the current closing bracket. This is exactly what a stack provides — last-in-first-out (LIFO) access.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Single pass" spaceNote="Stack in worst case" />

### Step-by-step Example

**Input:** `s = "([{}])"`

| Step | Char | Stack (before) | Action | Stack (after) |
| --- | --- | --- | --- | --- |
| 1 | `(` | `[]` | Opening bracket → push | `['(']` |
| 2 | `[` | `['(']` | Opening bracket → push | `['(', '[']` |
| 3 | `{` | `['(', '[']` | Opening bracket → push | `['(', '[', '{']` |
| 4 | `}` | `['(', '[', '{']` | Closing bracket → top is `{` → match! pop | `['(', '[']` |
| 5 | `]` | `['(', '[']` | Closing bracket → top is `[` → match! pop | `['(']` |
| 6 | `)` | `['(']` | Closing bracket → top is `(` → match! pop | `[]` |
| End | — | `[]` | Stack empty → valid ✓ | — |

**Input:** `s = "([)]"`

| Step | Char | Stack (before) | Action | Result |
| --- | --- | --- | --- | --- |
| 1 | `(` | `[]` | Opening bracket → push | `['(']` |
| 2 | `[` | `['(']` | Opening bracket → push | `['(', '[']` |
| 3 | `)` | `['(', '[']` | Closing bracket → top is `[`, but we see `)` → no match! ✗ | Invalid |

### Pseudocode

```plaintext
function validParenthesesStack(s):
    stack = []
    bracketMap = {'(': ')', '[': ']', '{': '}'}

    for char in s:
        if char is an opening bracket:
            stack.push(char)
        else:
            if stack is empty or bracketMap[stack.pop()] != char:
                return false
    
    return stack is empty
```

## Approach 2: Replace Loop

<ApproachTag tag="simple" />

Repeatedly remove consecutive pairs of matching brackets until no more can be removed. If the string becomes empty, it is valid.

This approach is conceptually simpler but less efficient. It directly models the idea that valid brackets always have matching pairs next to each other.

<Aside type="caution" title="Why this is slower">
Each removal operation scans the entire remaining string. In the worst case (e.g., `"((()))"`) you perform O(n) iterations, each taking O(n) time to remove pairs, giving O(n²) total time.
</Aside>

<ComplexityBadges time="O(n²)" space="O(n)" timeColor="red" spaceColor="yellow" timeNote="Multiple passes" spaceNote="Temporary strings" />

### Step-by-step Example

**Input:** `s = "([{}])"`

| Pass | Current String | Pairs Removed | Next String |
| --- | --- | --- | --- |
| 1 | `"([{}])"` | `{}` | `"([])"` |
| 2 | `"([])"` | `[]` | `"()"` |
| 3 | `"()"` | `()` | `""` |
| End | `""` | — | Valid ✓ |

**Input:** `s = "([)]"`

| Pass | Current String | Pairs Removed | Next String |
| --- | --- | --- | --- |
| 1 | `"([)]"` | — | `"([)]"` |
| End | `"([)]"` | No pairs to remove | Invalid ✗ |

### Pseudocode

```plaintext
function validParenthesesReplace(s):
    while s contains "()" or "[]" or "{}":
        remove all occurrences of "()", "[]", "{}"
    return s is empty
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting to check if the stack is empty** before popping when encountering a closing bracket. A closing bracket with no matching opening bracket should return false immediately.
- **Wrong bracket type match** — `"(]"` should return false because the closing bracket type does not match the opening bracket type.
- **Forgetting the final check** — even if you match all closing brackets, you must ensure the stack is empty (no unmatched opening brackets remain).
- **Checking stack non-empty after the loop** — common mistake: only checking `stack.empty()` but never validating matches during iteration.
</Aside>

## Interview Tips

<Aside type="tip" title="Key discussion points">
- **Why stack over brute force?** Stack processes the string in a single pass. The replace loop must repeatedly scan and modify the string, making it quadratic.
- **Follow-up: Generate parentheses** — given n pairs, generate all valid combinations. This requires backtracking and builds on the validation concept.
- **Follow-up: Longest valid parentheses** — find the longest substring of valid parentheses. This uses dynamic programming or stack with index tracking.
- **Edge cases to mention:** empty string (valid), single character (invalid), all opening brackets, all closing brackets, alternating mismatched types like `"(][)]"`.
- **Time vs. space trade-off:** Stack approach uses linear space to achieve linear time. There's no way to do this in O(1) space without exponential time.
</Aside>

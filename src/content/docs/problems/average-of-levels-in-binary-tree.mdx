## Problem Statement

Given the `root` of a binary tree, return an array of the average value of the nodes on each level of the tree.

Answers within 10^-5 of the actual answer will be accepted as correct.

<Aside type="tip" title="What to notice first">
- You need to process each level and calculate the average of all nodes in that level.
- Both BFS and DFS can solve this, but with different approaches.
- Be careful with floating-point precision when calculating averages.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[3,9,20,null,null,15,7]` | `[3.00000,14.50000,7.50000]` | Level 1: 3/1=3; Level 2: (9+20)/2=14.5; Level 3: (15+7)/2=7.5 |
| `[3,9,20]` | `[3.00000,14.50000]` | Level 1: 3; Level 2: (9+20)/2=14.5 |

## Constraints

- The number of nodes in the tree is in the range `[1, 10^4]`.
- `-2^31 <= Node.val <= 2^31 - 1`

## Approach 1: BFS Level-Order (Recommended)

<ApproachTag tag="recommended" />

Use a queue to traverse level by level. For each level, sum all node values, then divide by the count.

<ComplexityBadges time="O(n)" space="O(w)" timeColor="green" spaceColor="yellow" timeNote="Visit all nodes" spaceNote="Queue width at widest level" />

### Step-by-step Example

**Input:** `root = [3,9,20,null,null,15,7]`

Level 1: sum=3, count=1, avg=3.0
Level 2: sum=29, count=2, avg=14.5
Level 3: sum=22, count=2, avg=7.5

### Pseudocode

```plaintext
function averageOfLevelsBFS(root):
    result = []
    queue = [root]
    
    while queue is not empty:
        levelSize = length of queue
        levelSum = 0
        
        for i = 0 to levelSize - 1:
            node = queue.dequeue()
            levelSum += node.val
            
            if node.left exists:
                queue.enqueue(node.left)
            if node.right exists:
                queue.enqueue(node.right)
        
        result.append(levelSum / levelSize)
    
    return result
```

## Approach 2: DFS with Level Tracking

<ApproachTag tag="interview" />

Use DFS to traverse the tree, tracking the current level. Maintain lists of sums and counts per level, then calculate averages.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Visit all nodes" spaceNote="Recursion call stack" />

### Step-by-step Example

DFS visits nodes in pre-order: 3 (L0), 9 (L1), 20 (L1), 15 (L2), 7 (L2)

Accumulate:
- Level 0: sum=3, count=1
- Level 1: sum=29, count=2
- Level 2: sum=22, count=2

Average: [3.0, 14.5, 7.5]

### Pseudocode

```plaintext
function averageOfLevelsDFS(root):
    sums = []
    counts = []
    
    function dfs(node, level):
        if node is null:
            return
        
        if level == length of sums:
            sums.append(0)
            counts.append(0)
        
        sums[level] += node.val
        counts[level] += 1
        
        dfs(node.left, level + 1)
        dfs(node.right, level + 1)
    
    dfs(root, 0)
    
    result = []
    for i in range(length of sums):
        result.append(sums[i] / counts[i])
    
    return result
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Integer division instead of floating-point.** Use double/float for the average calculation.
- **Forgetting to initialize sum/count arrays.** Make sure you create them as you encounter new levels.
- **Off-by-one in level tracking.** Ensure levels are indexed correctly.
- **Not handling empty trees.** Return an empty list for null input.
</Aside>

## Problem Statement

Given the root of a complete binary tree, return the number of the nodes in the tree.

A complete binary tree is a binary tree in which every level, except possibly the last, is completely filled, and all nodes in the last level are as far left as possible.

<Aside type="tip" title="What to notice first">
- Complete tree property: last level filled left-to-right, all other levels complete.
- If left subtree height == right subtree height, left is perfect: count = 2^(h+1) - 1.
- Otherwise, right is perfect: count = 2^h - 1.
- Both approaches exploit this property to achieve O(log² n).
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[1,2,3,4,5,6]` | 6 | All 6 nodes present in complete tree |
| `[1,2,3,4,5,6,7]` | 7 | Perfect binary tree of height 2 |

## Constraints

- The number of nodes is in the range `[1, 5 * 10^4]`.
- `0 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">

## Approach 1: Level Calculation

<ApproachTag tag="recommended" />

Calculate left and right subtree heights. If equal, left is perfect; otherwise, right is perfect. Use formula 2^h - 1 for perfect subtrees.

<ComplexityBadges time="O(log² n)" space="O(log n)" timeColor="green" spaceColor="green" timeNote="Recursive depth × height calculation" spaceNote="Recursion stack" />

### Pseudocode

```plaintext
function countNodes(root):
    if not root: return 0
    
    leftHeight = countLeftDepth(root)
    rightHeight = countRightDepth(root)
    
    if leftHeight == rightHeight:
        return (2^(leftHeight+1) - 1) + countNodes(root.right)
    else:
        return (2^(rightHeight+1) - 1) + countNodes(root.left)
```

## Approach 2: Binary Search on Positions

<ApproachTag tag="alternative" />

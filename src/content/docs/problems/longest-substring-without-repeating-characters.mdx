## Problem Statement

Given a string `s`, find the length of the **longest substring** without repeating characters.

A substring is a contiguous sequence of characters within the string. You must return the length (integer), not the substring itself.

<Aside type="tip" title="What to notice first">
- You need to find a **contiguous** sequence, not just any selection of characters.
- The answer is the **length**, not the substring.
- The sliding window pattern is the gold standard here: move the right pointer forward and shrink from the left when a duplicate appears.
- A hash map or set to track which characters are in the current window is essential.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `"abcabcbb"` | `3` | The longest substring is `"abc"`, which has length 3. |
| `"bbbbb"` | `1` | The longest substring is `"b"`, which has length 1. |
| `"pwwkew"` | `3` | The longest substring is `"wke"` or `"kew"`, which has length 3. |
| `""` | `0` | Empty string has no substring. |
| `"au"` | `2` | The longest substring is `"au"`, which has length 2. |

## Constraints

- `0 <= s.length <= 5 * 10^4`
- `s` consists of English letters, digits, symbols, and spaces.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Password validation** — ensure a user's password has a minimum number of consecutive unique characters without repetition.
- **Data deduplication** — find the longest run of unique identifiers in a stream of log entries.
- **Session token parsing** — extract the longest valid session segment where no character repeats.
- **DNA sequence analysis** — identify the longest stretch of consecutive unique bases (A, T, G, C) in genomic data.
- **Token/identifier validation** — verify that API rate limits or access tokens contain a unique substring of sufficient length.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Returns | Best When |
| --- | --- | --- | --- | --- |
| Brute Force | O(n³) | O(min(n, m)) | Length | String is tiny, practicing the concept, or interviewer explicitly requests it |
| Sliding Window | O(n) | O(min(n, m))\* | Length | General case — fast, clean, and efficient |

\* Space is proportional to the character set size, not string length. For English letters only, this is O(26) = O(1).

- Pick **Sliding Window** if you want the best all-around interview answer.
- Pick **Brute Force** if you are still learning how to think about substrings.
- **Every interviewer expects the sliding window solution** for this problem.

## Approach 1: Sliding Window with Hash Map (Recommended)

<ApproachTag tag="recommended" />

Use two pointers (`left` and `right`) to maintain a window. Expand the window by moving `right` forward. When a character is seen that already exists in the current window, shrink the window from the left until the duplicate is removed.

The key insight is that each character is visited at most twice (once by `right`, once by `left`), making this a single linear pass through the string.

<ComplexityBadges time="O(n)" space="O(min(n, m))" timeColor="green" spaceColor="green" timeNote="Each character visited once per pointer" spaceNote="Character set (max 26 letters or full Unicode)" />

### Pseudocode

```plaintext
function longestSubstringWithoutRepeatingCharactersSlidingWindow(s):
    charIndex = {}
    maxLength = 0
    left = 0

    for right from 0 to len(s) - 1:
        if s[right] in charIndex and charIndex[s[right]] >= left:
            left = charIndex[s[right]] + 1
        charIndex[s[right]] = right
        maxLength = max(maxLength, right - left + 1)

    return maxLength
```

## Approach 2: Brute Force

<ApproachTag tag="simple" />

Check every possible substring and find the longest one without repeating characters.

For each starting position `i`, extend to position `j` and check if all characters from `i` to `j` are unique. Keep track of the maximum length found.

This approach is slow but instructive: it explicitly shows what a "valid substring" is.

<ComplexityBadges time="O(n³)" space="O(min(n, m))" timeColor="red" spaceColor="green" timeNote="Three nested loops: i, j, and checking duplicates" spaceNote="Set for characters in current window" />

### Step-by-step Example

**Input:** `s = "abcabcbb"`

| i | j | substring | has duplicate? | action | max_length |
| --- | --- | --- | --- | --- | --- |
| 0 | 0 | "a" | No | Add to set, update max | 1 |
| 0 | 1 | "ab" | No | Add 'b', update max | 2 |
| 0 | 2 | "abc" | No | Add 'c', update max | 3 |
| 0 | 3 | "abca" | Yes ('a') | Break inner loop | 3 |
| 1 | 1 | "b" | No | Start new substring | 3 |
| 1 | 2 | "bc" | No | Add 'c', max stays 3 | 3 |
| 1 | 3 | "bca" | No | Add 'a', max stays 3 | 3 |
| 1 | 4 | "bcab" | Yes ('b') | Break | 3 |

### Pseudocode

```plaintext
function longestSubstringWithoutRepeatingCharactersBruteForce(s):
    maxLength = 0

    for i from 0 to len(s) - 1:
        charSet = {}
        for j from i to len(s) - 1:
            if s[j] in charSet:
                break
            charSet.add(s[j])
            maxLength = max(maxLength, j - i + 1)

    return maxLength
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Returning the substring instead of the length.** The problem asks for the length as an integer, not the substring.
- **Forgetting to move the left pointer when a duplicate is found.** The window must shrink to exclude the duplicate.
- **Off-by-one errors with window size.** The window size is `right - left + 1`, not `right - left`.
- **Not updating the character's position in the hash map.** Always update after moving `left`, or the next check will reference a stale index.
- **Using a set instead of tracking indices.** A set works but doesn't tell you where the duplicate is, forcing extra work to shrink the window correctly.
- **Assuming ASCII characters only.** Unicode characters, digits, and symbols are valid and should be handled the same way.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why sliding window over brute force?** A dramatic drop from O(n³) to O(n). The sliding window is the standard for any "longest/shortest substring" problem.
- **How do you know when to use sliding window?** Look for the key pattern: you want a contiguous element (substring, subarray) that satisfies a property. Sliding window is almost always the answer.
- **Edge cases to mention:** empty string (answer is 0), single character (answer is 1), all unique characters (answer is the string length), all duplicates like "aaaa" (answer is 1).
- **Follow-up questions you should be ready for:**
  - Return the actual substring, not just the length → track start and end indices.
  - Handle multiple character sets (Unicode) → the approach is identical; just ensure your hash map supports any character type.
  - Find all substrings of the maximum length → collect them as you track the max.
- **Common interviewer follow-up:** "Can you do this with O(1) space if the string contains only lowercase English letters?" Answer: Yes, use a 26-element array instead of a hash map.
</Aside>

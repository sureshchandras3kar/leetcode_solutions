## Problem Statement

Determine if a 9 × 9 Sudoku board is valid. Only the filled cells need to be validated according to the following rules:

- Each **row** must contain the digits 1–9 without repetition.
- Each **column** must contain the digits 1–9 without repetition.
- Each of the nine **3×3 sub-boxes** of the grid must contain the digits 1–9 without repetition.

**Note:** A Sudoku board (partially filled) could be valid but is not necessarily solvable. Only the filled cells need to be validated.

<Aside type="tip" title="What to notice first">
- The hardest part is the box index formula: `box = (row // 3) * 3 + (col // 3)`
- `'.'` represents an empty cell — skip those entirely.
- This is a **validation** problem, not a solving problem — you only check that no digit appears twice, not that a solution exists.
</Aside>

### Examples

| Board | Valid? | Reason |
| --- | --- | --- |
| Standard partially filled valid board | ✓ | No duplicates in any row, column, or box |
| Board with `'8'` repeated in the same row | ✗ | Duplicate `8` in row 0 |

**Valid board (first 3 rows shown):**

```
["5","3",".",".","7",".",".",".","."]
["6",".",".","1","9","5",".",".","."]
[".","9","8",".",".",".",".","6","."]
```

No digit repeats in any row, column, or 3×3 box — valid.

**Invalid board:**

```
["8","8",".",".","7",".",".",".","."]
...
```

`'8'` appears twice in row 0 — invalid.

## Constraints

- `board.length == 9`
- `board[i].length == 9`
- `board[i][j]` is a digit `'1'`–`'9'` or `'.'`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Constraint satisfaction validation** — checking that a partially completed puzzle satisfies all constraints before solving.
- **Spreadsheet validation** — detecting conflicts in constraint-based cell value assignments.
- **Exam seating arrangements** — verifying no two students with the same topic are in the same row, column, or block.
- **Resource allocation** — ensuring no two processes require the same resource in overlapping time slots.
</Aside>

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Hash Sets | O(1) | O(1) | Board is always 9×9 — both are constant |

Both are O(1) since the board size is fixed at 9×9.

## Approach 1: Hash Sets

<ApproachTag tag="recommended" />

Use three arrays of 9 sets — `rows[9]`, `cols[9]`, `boxes[9]` — to track which digits have been seen. For every non-empty cell `(r, c)` with digit `d`:

1. Compute `box = (r // 3) * 3 + (c // 3)`
2. If `d` is already in `rows[r]`, `cols[c]`, or `boxes[box]` → return `false`
3. Otherwise add `d` to all three sets

If no conflict is found after all 81 cells → return `true`.

<ComplexityBadges time="O(1)" space="O(1)" timeColor="green" spaceColor="green" timeNote="81 cells fixed" spaceNote="27 sets of ≤9 digits" />

### Box Index Derivation

The 9 boxes are numbered 0–8 left-to-right, top-to-bottom:

| rows \ cols | 0–2 | 3–5 | 6–8 |
| --- | --- | --- | --- |
| 0–2 | box 0 | box 1 | box 2 |
| 3–5 | box 3 | box 4 | box 5 |
| 6–8 | box 6 | box 7 | box 8 |

Formula: `box = (row // 3) * 3 + (col // 3)`

### Step-by-step: Processing row 0

**Sample row 0:** `["5","3",".",".","7",".",".",".","."]`

| Step | (r, c) | d | box | rows[0] after | cols[c] after | boxes[box] after |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | (0, 0) | 5 | 0 | `{5}` | `{5}` | `{5}` |
| 2 | (0, 1) | 3 | 0 | `{5,3}` | `{3}` | `{5,3}` |
| 3 | (0, 2) | . | — | skip | skip | skip |
| 4 | (0, 3) | . | — | skip | skip | skip |
| 5 | (0, 4) | 7 | 1 | `{5,3,7}` | `{7}` | `{7}` |
| … | … | . | — | skip | skip | skip |

### Pseudocode

```plaintext
function isValidSudoku(board):
    rows  = 9 empty sets
    cols  = 9 empty sets
    boxes = 9 empty sets
    for r in 0..8:
        for c in 0..8:
            d = board[r][c]
            if d == '.': continue
            box = (r // 3) * 3 + (c // 3)
            if d in rows[r] or d in cols[c] or d in boxes[box]:
                return false
            add d to rows[r], cols[c], boxes[box]
    return true
```

## Approach 2: Bitmask

<ApproachTag tag="interview" />

Replace each set with a single integer. Bit `k` (0-indexed) is set when digit `k+1` has been seen. This eliminates hash overhead and reduces each check to a bitwise AND.

- **Check** digit `d`: `(mask >> (d - 1)) & 1 == 1`
- **Set** digit `d`: `mask |= (1 << (d - 1))`

<ComplexityBadges time="O(1)" space="O(1)" timeColor="green" spaceColor="green" timeNote="81 cells fixed" spaceNote="27 integers" />

### Bitmask Example

For digit `5`: bit index = `5 - 1 = 4`, so bit pattern = `0b000010000` (bit 4 set).

| Digit seen | Bit set | mask value |
| --- | --- | --- |
| 5 | bit 4 | `0b000010000` = 16 |
| 3 | bit 2 | `0b000010100` = 20 |
| 7 | bit 6 | `0b001010100` = 84 |

To check if `3` is already in this mask: `84 & (1 << 2)` = `84 & 4` = `4` ≠ 0 → duplicate detected.

### Pseudocode

```plaintext
function isValidSudoku(board):
    rows  = [0] * 9
    cols  = [0] * 9
    boxes = [0] * 9
    for r in 0..8:
        for c in 0..8:
            d = board[r][c]
            if d == '.': continue
            bit = 1 << (int(d) - 1)
            box = (r // 3) * 3 + (c // 3)
            if rows[r] & bit or cols[c] & bit or boxes[box] & bit:
                return false
            rows[r]  |= bit
            cols[c]  |= bit
            boxes[box] |= bit
    return true
```

<Aside type="caution" title="Common mistakes">
- **Incorrect box index formula** — the correct formula is `(row // 3) * 3 + (col // 3)`, not `row * 3 + col` or any other variant.
- **Forgetting to skip `'.'` cells** — attempting to insert or check `'.'` corrupts all three sets/masks.
- **Thinking completeness is required** — you only need to confirm no digit appears twice among the filled cells; an incomplete board can still be valid.
</Aside>

## Interview Tips

<Aside type="tip" title="Key points to communicate">
- **State the box formula immediately** — `(row // 3) * 3 + (col // 3)` is the one part that trips people up, so say it out loud early.
- **Both approaches are O(1) time and space** because the board is fixed-size (9×9) — the constant 81 cells and 27 tracking structures never grow.
- **The bitmask approach is a clean follow-up** — "I can do this with 27 integers instead of 27 sets, replacing hash operations with bitwise AND/OR."
- **Practice tracing the box index** — verify: (0,0)→0, (0,3)→1, (3,0)→3, (4,4)→4, (8,8)→8.
</Aside>

## Problem Statement

Implement a **Trie** (also called prefix tree) with the following operations:
- `insert(word)` — Insert a word into the trie.
- `search(word)` — Search for an exact word in the trie and return whether it exists.
- `startsWith(prefix)` — Return whether there is any word in the trie that starts with the given prefix.

<Aside type="tip" title="What to notice first">
- A Trie efficiently stores and retrieves strings by their prefixes.
- Each node can have up to 26 children (for lowercase English letters).
- The key insight is: a prefix doesn't need to be a complete word.
- Both search and startsWith traverse the same path, differing only in their end-of-word check.
</Aside>

### Examples

| Operation | Input | Output | Notes |
| --- | --- | --- | --- |
| Insert | `"apple"` | — | Tree now has path: a→p→p→l→e |
| Search | `"apple"` | `true` | Complete word exists |
| Search | `"app"` | `false` | Prefix exists but not marked as word |
| StartsWith | `"app"` | `true` | Prefix exists in tree |
| Insert | `"app"` | — | Now search("app") returns true |

## Constraints

- `1 <= word.length, prefix.length <= 2000`
- `word` and `prefix` consist only of lowercase English letters.
- At most `3 × 10^4` calls will be made to `insert`, `search`, and `startsWith`.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Autocomplete systems** — quickly find all words with a given prefix.
- **Spell checkers** — validate words and suggest corrections by prefix.
- **IP routing tables** — organize network prefixes for fast packet routing.
- **Dictionary compression** — efficiently store and retrieve large word lists.
- **Search engine indexing** — prefix matching for search suggestions.
</Aside>

## Complexity Comparison

| Approach | Insert | Search | StartsWith | Space |
| --- | --- | --- | --- | --- |
| Recursive | O(m) | O(m) | O(m) | O(m × 26) |
| Iterative | O(m) | O(m) | O(m) | O(m × 26) |

*m = length of word/prefix, space = nodes × alphabet size*

## Which approach should you learn first?

- Pick **Iterative** for clarity and interviews — easier to explain and debug.
- Pick **Recursive** if you want to practice the recursive tree traversal pattern.

## Approach 1: Iterative (Recommended)

<ApproachTag tag="recommended" />

Build a tree of nodes where each node represents a character. For each operation, iterate through the characters, creating new nodes as needed during insertion. For search and prefix matching, follow the existing path and return whether it reaches the target or beyond.

<ComplexityBadges time="O(m)" space="O(m×26)" timeColor="green" spaceColor="yellow" timeNote="Single pass per word" spaceNote="Trie nodes storage" />

### Step-by-step Example

**Insert "cat":**

```
Insert 'c': root → c (new node)
Insert 'a': c → a (new node)
Insert 't': a → t (new node, mark as end-of-word)
```

**Search "cat":**

```
Follow 'c' → 'a' → 't' → found and marked as end-of-word → return true
```

**Search "ca":**

```
Follow 'c' → 'a' → path exists but NOT marked as end-of-word → return false
```

**StartsWith "ca":**

```
Follow 'c' → 'a' → path exists → return true (doesn't care about end-of-word)
```

### Pseudocode

```plaintext
class TrieNode:
    children = {}
    is_end_of_word = False

class Trie:
    root = TrieNode()

    function insert(word):
        node = root
        for each char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end_of_word = True

    function search(word):
        node = root
        for each char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end_of_word

    function startsWith(prefix):
        node = root
        for each char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to mark the last node as `is_end_of_word` during insertion.
- Returning `True` in `search` when the path exists but the word is not complete.
- Mixing up `search` (must be exact word) and `startsWith` (prefix only).
- In languages like Java, forgetting to initialize the HashMap in each node.
- Off-by-one errors when iterating through characters.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Iterative vs. Recursive:** Iterative is clearer and avoids stack overflow; recursive uses helper methods to reduce duplication.
- **Space optimization:** Using arrays instead of HashMaps for children (fixed size 26) saves space at the cost of memory per node.
- **Extensions:** Discuss how to add deletion, counting words, or returning all words with a prefix.
- **Follow-up:** How would you handle non-English characters? (Use larger alphabet or Unicode maps instead of fixed 26.)
- **Edge cases:** Empty strings, single character words, and duplicate inserts.
</Aside>

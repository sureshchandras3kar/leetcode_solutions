## Problem Statement

Given an `m x n` board containing `'X'` and `'O'`, capture all regions of `'O'` that are surrounded by `'X'`.

A region is captured by flipping all `'O'`s into `'X'`s in that surrounded region.

<Aside type="tip" title="What to notice first">
- Only `'O'` regions that are NOT connected to the border can be captured.
- Use DFS/BFS from border to mark all border-connected `'O'`s as safe.
- Flip remaining unmarked `'O'`s to `'X'`.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[["X","X","X","X"],["X","O","O","X"],["X","X","O","X"],["X","O","X","X"]]` | `[["X","X","X","X"],["X","X","X","X"],["X","X","X","X"],["X","O","X","X"]]` | Only the bottom-left O is surrounded |

## Constraints

- `m == board.length`
- `n == board[i].length`
- `1 <= m, n <= 200`
- `board[i][j]` is `'X'` or `'O'`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Game development** — flood fill with obstacles (Minesweeper).
- **Image processing** — connected component analysis with boundaries.
- **Geographic analysis** — identifying enclosed regions.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Notes |
| --- | --- | --- | --- |
| DFS | O(m * n) | O(m * n) | Recursive exploration, intuitive |
| Union-Find | O(m * n * α(n)) | O(m * n) | Disjoint sets, path compression |

## Which approach should you learn first?

- Pick **DFS** for most intuitive and straightforward solution.
- Pick **Union-Find** to understand disjoint set data structure.

## Approach 1: DFS — Recommended

<ApproachTag tag="recommended" />

Mark all border-connected 'O' regions from all four borders using DFS, then flip remaining 'O's to 'X's.

<ComplexityBadges time="O(m * n)" space="O(m * n)" timeColor="green" spaceColor="yellow" timeNote="Visit each cell once" spaceNote="Recursion stack" />

### Pseudocode

```plaintext
function solve(board):
    rows = len(board)
    cols = len(board[0])
    
    // Mark border-connected O's from all borders
    for each cell on border:
        if cell is 'O':
            dfs(board, r, c, rows, cols)
    
    // Flip unmarked O's to X's
    for each cell in board:
        if cell is 'O':
            cell = 'X'
        else if cell is '#':  // Marked O's
            cell = 'O'
```

## Approach 2: Union-Find

<ApproachTag tag="simple" />

Create a virtual boundary node and union it with all border-connected 'O's. Flip 'O's that are not connected to boundary.

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **DFS vs Union-Find:** DFS is simpler but uses implicit stack; Union-Find is more elegant for understanding disjoint sets.
- **Border strategy:** Key insight is to work from borders outward rather than interiors.
- **Follow-up:** Can you do this without modifying the board? Use a visited set instead.
</Aside>

## Problem Statement

You are a professional robber planning to rob houses along a street. Each house has a certain amount of money stashed. Robbers have a constraint: you cannot rob two adjacent houses.

Given an integer array `nums` representing the amount of money in each house, return the maximum amount of money you can rob without robbing adjacent houses.

<Aside type="tip" title="What to notice first">
- You cannot rob two adjacent houses in a single robbery.
- You want to maximize the total amount of money stolen.
- Every house is independent: robbing one doesn't affect the value of another, only the adjacency rule applies.
- This is a classic DP problem where each house presents a choice: rob or skip.
</Aside>

### Examples

| nums | Output | Explanation |
| --- | --- | --- |
| `[1, 2, 3, 1]` | `4` | Rob house 0 (money = 1) and house 2 (money = 3). Total = 1 + 3 = 4 |
| `[2, 7, 9, 3, 1]` | `12` | Rob house 1 (money = 7) and house 3 (money = 3). Total = 7 + 9 = 12 |
| `[5, 3, 4, 11, 2]` | `16` | Rob houses 0, 2, 4. Total = 5 + 4 + 11 = 20... wait, that's 20. Let me recalculate: houses 0 (5) + 2 (4) + 4 (2) = 11, or 0 (5) + 3 (11) = 16 ✓ |

## Constraints

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 400`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Financial portfolio optimization** — selecting non-adjacent stocks to maximize returns while managing risk.
- **Work scheduling** — choosing non-consecutive shifts to maximize pay while respecting rest periods.
- **Route planning** — selecting non-adjacent waypoints in a city to maximize resources collected.
- **Product bundle selection** — choosing complementary (non-competing) products to maximize revenue.
</Aside>

## Approach 1: DP Array

<ApproachTag tag="recommended" />

Create a DP array where `dp[i]` represents the maximum money you can rob from houses 0 to i.

For each house i, you have two choices:
- **Rob house i**: take `nums[i] + dp[i-2]` (the money from this house plus the max from non-adjacent houses before it)
- **Skip house i**: take `dp[i-1]` (the max from the house before it)

The recurrence is: `dp[i] = max(nums[i] + dp[i-2], dp[i-1])`

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Single pass" spaceNote="DP array storage" />

### Step-by-step Example

**Input:** `nums = [2, 7, 9, 3, 1]`

| i | nums[i] | Choice (Rob) | Choice (Skip) | dp[i] | Explanation |
| --- | --- | --- | --- | --- | --- |
| 0 | 2 | 2 | 0 | 2 | Rob house 0 |
| 1 | 7 | 7 | 2 | 7 | Rob house 1 (better than house 0) |
| 2 | 9 | 9 + 2 = 11 | 7 | 11 | Rob house 2 and non-adjacent house 0 |
| 3 | 3 | 3 + 7 = 10 | 11 | 11 | Skip house 3, keep previous max |
| 4 | 1 | 1 + 11 = 12 | 11 | 12 | Rob house 4 with best from 0-2 |

**Result:** Max money = 12 (houses 1 and 3, or houses 0 and 2)

### Pseudocode

```plaintext
function houseRobberDpArray(nums):
    if nums is empty:
        return 0
    if nums length is 1:
        return nums[0]
    
    dp = array of size len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    
    for i from 2 to len(nums) - 1:
        dp[i] = max(nums[i] + dp[i-2], dp[i-1])
    
    return dp[len(nums) - 1]
```

## Approach 2: Space-Optimized

<ApproachTag tag="interview" />

Instead of storing the entire DP array, observe that `dp[i]` only depends on `dp[i-1]` and `dp[i-2]`. Therefore, we only need two variables: `prev1` (max money up to house i-1) and `prev2` (max money up to house i-2).

This approach reduces space complexity from O(n) to O(1) while maintaining the same time complexity.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceNote="Only two variables" />

### Step-by-step Example

**Input:** `nums = [2, 7, 9, 3, 1]`

| i | nums[i] | prev2 | prev1 | current | Update |
| --- | --- | --- | --- | --- | --- |
| Start | — | 2 | 7 | — | Initialize with first two houses |
| 2 | 9 | 2 | 7 | 11 | max(9+2, 7) = 11, shift: prev2=7, prev1=11 |
| 3 | 3 | 7 | 11 | 11 | max(3+7, 11) = 11, shift: prev2=11, prev1=11 |
| 4 | 1 | 11 | 11 | 12 | max(1+11, 11) = 12 |

**Result:** Max money = 12

### Pseudocode

```plaintext
function houseRobberSpaceOptimized(nums):
    if nums is empty:
        return 0
    if nums length is 1:
        return nums[0]
    
    prev2 = nums[0]
    prev1 = max(nums[0], nums[1])
    
    for i from 2 to len(nums) - 1:
        current = max(nums[i] + prev2, prev1)
        prev2 = prev1
        prev1 = current
    
    return prev1
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting the adjacency constraint:** ensure you skip the house immediately before the one you rob.
- **Base case errors:** dp[1] should be the max of the first two houses, not just nums[1].
- **Variable update order:** in the space-optimized approach, update prev2 before prev1, or use a temporary variable.
- **Off-by-one errors:** the loop should iterate from index 2 to len(nums) - 1 (inclusive).
- **Empty or single-house arrays:** handle edge cases where nums.length <= 1.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why DP over brute force?** Brute force checks all 2^n subsets. DP uses the optimal substructure to reduce to O(n).
- **State representation:** clearly explain what dp[i] represents and why only the last two values matter.
- **Follow-up: House Robber II (circular):** the same DP logic applies, but you handle the circular constraint by running two DP passes (excluding the first or last house).
- **Edge cases:** mention handling nums with 0 or 1 elements.
- **Proof of correctness:** the greedy property holds: at each house, the optimal choice is always to pick the maximum of robbing or skipping.
</Aside>

## Problem Statement

Given a binary tree, find the **lowest common ancestor (LCA)** of two given nodes `p` and `q` in the tree.

The lowest common ancestor is the deepest node in the tree that is an ancestor of both `p` and `q`. A node can be an ancestor of itself.

<Aside type="tip" title="What to notice first">
- A node can be an ancestor of itself (if one of p or q is the ancestor of the other).
- The LCA is the deepest node that has both p and q in its subtree.
- Two different approaches: one uses recursion (DFS), the other uses parent pointers with hash set.
</Aside>

### Examples

| Input | p | q | Output | Explanation |
| --- | --- | --- | --- | --- |
| `[3,5,1,6,2,0,8,null,null,7,4]` | `5` | `1` | `3` | 3 is the LCA of 5 and 1 |
| `[3,5,1,6,2,0,8,null,null,7,4]` | `5` | `4` | `5` | 5 is the LCA of 5 and 4 (5 is ancestor of 4) |
| `[1,2]` | `1` | `2` | `1` | 1 is the LCA of 1 and 2 |

### Visualized Examples

```

Example 3:       1        LCA(1, 2) = 1 (1 is ancestor of itself and 2)

```

## Constraints

- The number of nodes in the tree is in the range `[2, 10^5]`.
- `-10^9 <= Node.val <= 10^9`
- All nodes have unique values.
- p and q will exist in the tree.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **File system hierarchies** — find the common parent directory of two files.
- **Organizational structures** — find the lowest common manager of two employees.
- **Git commit history** — find the merge base of two branches.
- **DNA sequence analysis** — find the common ancestor in evolutionary trees.
- **Social networks** — find the closest common connection between two users.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| DFS Recursive | O(n) | O(h) | Simple to implement; space is the call stack |
| Parent Pointers + Hash Set | O(n) | O(h) | If multiple LCA queries needed; explicit space usage |

**Where h = height (at most n)**

## Which approach should you learn first?

- Pick **DFS Recursive** if you want the simplest, most elegant solution that naturally leverages tree structure.
- Pick **Parent Pointers + Hash Set** if you want to understand how to track ancestry and use hash sets for quick lookups.

## Approach 1: DFS Recursive (Recommended)

<ApproachTag tag="recommended" />

The key insight: the LCA of p and q is the deepest node where p and q are in different subtrees (or one is the node itself).

For each node, recursively search for p and q in its left and right subtrees:
- If both are found in different subtrees, this node is the LCA.
- If both are in one subtree, recurse into that subtree.
- If found in a subtree, return that node to propagate up.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Visit all nodes in worst case" spaceNote="Recursion call stack" />

### Step-by-step Example

**Input:** Tree with nodes [3,5,1,6,2,0,8], p = 5, q = 1

```
       3
      / \
     5   1
    / \ / \
   6  2 0  8
```

| Node | Left Result | Right Result | Decision | Reason |
| --- | --- | --- | --- | --- |
| 6 | null | null | null | Leaf, neither p nor q |
| 2 | null | null | null | Leaf, neither p nor q |
| 5 | p found | 2 (not q) | return 5 | Left subtree has p |
| 0 | null | null | null | Leaf, neither p nor q |
| 8 | null | null | null | Leaf, neither p nor q |
| 1 | 0 (not p) | 8 (not q) | return 1 | This is q itself |
| 3 | 5 (has p) | 1 (has q) | **return 3** ✓ | Both in different subtrees |

### Pseudocode

```plaintext
function lca(root, p, q):
    if root is null:
        return null
    
    if root is p or root is q:
        return root
    
    left = lca(root.left, p, q)
    right = lca(root.right, p, q)
    
    if left is not null and right is not null:
        return root          // Both found in different subtrees
    
    return left if left is not null else right  // One subtree has both
```

## Approach 2: Parent Pointers + Hash Set

<ApproachTag tag="interview" />

Store parent pointers for each node during a DFS traversal. Then, for node p, collect all ancestors in a hash set. Walk up from q using parent pointers until finding a node in the set.

This approach separates the traversal (finding nodes and storing parents) from the LCA search.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="yellow" timeNote="One DFS to find parents, then walk ancestors" spaceNote="Hash set of ancestors" />

### Step-by-step Example

**Input:** Tree with nodes [3,5,1,6,2,0,8], p = 5, q = 1

Step 1: Find p and store all ancestors of p:
```
DFS traversal finds p=5
Ancestors of 5: {5, 3}  // 5 itself and its ancestor 3
```

Step 2: Walk from q upward, check each ancestor against the set:
```
Walk from q=1: 
  Check 1 → not in {5, 3}
  Check parent(1)=3 → found in set!
  LCA is 3 ✓
```

### Pseudocode

```plaintext
function lca(root, p, q):
    parent = {}
    visited = set()
    
    // DFS to store parent pointers
    function dfs(node):
        if node is null:
            return
        visited.add(node)
        if node.left is not null:
            parent[node.left] = node
            dfs(node.left)
        if node.right is not null:
            parent[node.right] = node
            dfs(node.right)
    
    dfs(root)
    
    // Collect all ancestors of p
    ancestors = set()
    current = p
    while current is not null:
        ancestors.add(current)
        current = parent.get(current)
    
    // Walk from q up to find first ancestor in set
    current = q
    while current not in ancestors:
        current = parent.get(current)
    
    return current
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting that a node can be an ancestor of itself.** If q is in p's subtree, p is the LCA.
- **Not handling null nodes properly.** A null node should return null, not cause an error.
- **Assuming the tree is a BST.** This solution works for any binary tree, not just BSTs.
- **Off-by-one logic in parent pointers.** Make sure you're checking ancestors correctly and not skipping the node itself.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **DFS vs Parent Pointers:** DFS is simpler and more elegant; parent pointers are more flexible if you need multiple LCA queries.
- **Single pass vs multiple passes:** DFS does it in one recursive pass; parent pointers require explicit parent storage.
- **Memory usage:** Both are O(h), but parent pointers use explicit dictionary space.
- **Follow-ups to expect:** handling BST version, handling null inputs, multiple nodes, or finding all LCAs.
</Aside>

## Problem Statement

Given the `root` of a binary tree, imagine yourself standing on the right side of it, return the values of the nodes you can see ordered from top to bottom.

<Aside type="tip" title="What to notice first">
- You see only the rightmost node at each level.
- If a right child exists, you see it; otherwise you see the leftmost node that extends to that level.
- Both BFS and DFS work, but they visit levels differently.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `[1,2,3,null,5,null,4]` | `[1,3,4]` | Right view: root 1, right child 3, right-most at level 3 is 4 |
| `[1,null,3]` | `[1,3]` | Only right children visible |
| `[]` | `[]` | Empty tree |

### Visualized Examples

Example 2:       1

## Constraints

- The number of nodes in the tree is in the range `[0, 100]`.
- `-100 <= Node.val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **3D rendering** — determine visible surfaces from a viewing angle.
- **Organizational charts** — view reporting lines from the side.
- **Route planning** — see landmarks from a specific viewpoint.
- **Game graphics** — render visible objects in a scene.
- **Photography** — identify prominent features in composition.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| BFS Level-Order | O(n) | O(w) | Want to process level-by-level clearly |
| DFS Right-First | O(n) | O(h) | Prefer simpler recursion; smaller trees |

**Where h = height, w = max width**

## Which approach should you learn first?

- Pick **BFS Level-Order** if you want a clear, intuitive level-by-level approach.
- Pick **DFS Right-First** if you prefer elegant recursion that processes the right side first.

## Approach 1: BFS Level-Order (Recommended)

<ApproachTag tag="recommended" />

Process the tree level by level. For each level, take the rightmost node (the last one dequeued in that level).

<ComplexityBadges time="O(n)" space="O(w)" timeColor="green" spaceColor="yellow" timeNote="Visit all nodes once" spaceNote="Queue width at widest level" />

### Step-by-step Example

**Input:** `root = [1,2,3,null,5,null,4]`

```
Level 1: [1]         → right-most = 1
Level 2: [2, 3]      → right-most = 3
Level 3: [5, 4]      → right-most = 4
Result: [1, 3, 4]
```

### Pseudocode

```plaintext
function rightSideViewBFS(root):
    if root is null:
        return []
    
    result = []
    queue = [root]
    
    while queue is not empty:
        levelSize = length of queue
        
        // Process all nodes at this level
        for i = 0 to levelSize - 1:
            node = queue.dequeue()
            
            // Store the rightmost (last) node at this level
            if i == levelSize - 1:
                result.append(node.val)
            
            if node.left exists:
                queue.enqueue(node.left)
            if node.right exists:
                queue.enqueue(node.right)
    
    return result
```

## Approach 2: DFS Right-First Traversal

<ApproachTag tag="interview" />

Use DFS and visit the right child before the left child. Track the current level. Record a node only the first time we visit its level (which is the rightmost at that level).

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Visit all nodes" spaceNote="Recursion call stack" />

### Step-by-step Example

**Input:** `root = [1,2,3,null,5,null,4]`

DFS order (right-first): 1 (level 0) → 3 (level 1) → 4 (level 2) → 2 (level 1, skip) → 5 (level 2, skip)

Result: [1, 3, 4]

### Pseudocode

```plaintext
function rightSideViewDFS(root):
    result = []
    
    function dfs(node, level):
        if node is null:
            return
        
        // First time reaching this level, it's the rightmost
        if level == length of result:
            result.append(node.val)
        
        // Right child first, then left
        dfs(node.right, level + 1)
        dfs(node.left, level + 1)
    
    dfs(root, 0)
    return result
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Taking the wrong node at each level.** Make sure you're getting the rightmost, not the leftmost.
- **DFS: visiting left before right.** In the DFS approach, visit right first so the first time you reach a level is from the right side.
- **Not handling empty trees.** Return an empty list for null input.
- **Off-by-one errors with level tracking.** Ensure your level tracking is correct.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **BFS vs DFS:** BFS is more intuitive for level processing; DFS is elegant but requires careful right-first ordering.
- **Space usage:** BFS uses queue space (width-dependent); DFS uses call stack (height-dependent).
- **Follow-ups to expect:** left side view, top view, bottom view, or per-level aggregations.
</Aside>

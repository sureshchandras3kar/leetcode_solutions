## Problem Statement

Given an unsorted integer array `nums`, return the **smallest missing positive integer**. You must implement an algorithm that runs in **O(n) time** and uses **O(1) auxiliary space**.

<Aside type="caution" title="Two hard constraints">
- O(n) time rules out sorting (O(n log n))
- O(1) space rules out a hash set
- The trick: use the input array itself as the hash table — it has n slots and the answer is in [1, n+1]
</Aside>

### Examples

| Input | Output | Why |
| --- | --- | --- |
| `[1,2,0]` | `3` | 1 and 2 present, 3 missing |
| `[3,4,-1,1]` | `2` | 1 present, 2 missing |
| `[7,8,9,11,12]` | `1` | 1 missing immediately |

## Constraints

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **ID gap detection** — find the first unused user/order/ticket ID in a dense sequence
- **Memory page allocation** — find the first unallocated page frame number
- **Port assignment** — find the first unused network port in a range
- **Sequence reconstruction** — find the first missing term in a recorded integer sequence
</Aside>

## Complexity Comparison

| Approach | Time | Space | Notes |
| --- | --- | --- | --- |
| Index as Hash | O(n) | O(1) | Uses negative marking; modifies input |
| Cycle Sort | O(n) | O(1) | Places elements at correct positions |
| Hash Set (not allowed) | O(n) | O(n) | Conceptually simple but violates constraint |

## Approach 1: Index as Hash (Recommended)

<ApproachTag tag="recommended" />

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Three linear passes" spaceNote="In-place marking" />

The key insight: the answer must be in the range [1, n+1]. This means the array's n slots are exactly enough to encode presence of every candidate value. We use **negative sign as a boolean flag** at the index corresponding to each value.

**Three phases:**
1. **Clean**: replace irrelevant values (≤0 or >n) with n+1 so they don't interfere with marking
2. **Mark**: for each value v = abs(nums[i]) where 1 ≤ v ≤ n, negate nums[v-1] to mark "v is present"
3. **Scan**: find first index i where nums[i] > 0 → return i+1

### Step-by-step Example

**Input:** `[3,4,-1,1]`, n = 4

**Phase 1** — replace -1 (≤0) with n+1=5: `[3,4,5,1]`

**Phase 2** — mark:

| i | abs(nums[i]) | mark index | array after |
| --- | --- | --- | --- |
| 0 | 3 | 2 | [3,4,-5,1] |
| 1 | 4 | 3 | [3,4,-5,-1] |
| 2 | 5 > n | skip | [3,4,-5,-1] |
| 3 | 1 | 0 | [-3,4,-5,-1] |

**Phase 3** — scan `[-3,4,-5,-1]`: index 1 has positive 4 → **return 2** ✓

### Pseudocode

```plaintext
function firstMissingPositive(nums):
    n = len(nums)
    # Phase 1: replace non-positives and > n
    for i in range(n):
        if nums[i] <= 0 or nums[i] > n:
            nums[i] = n + 1
    # Phase 2: mark presence
    for i in range(n):
        v = abs(nums[i])
        if 1 <= v <= n:
            nums[v-1] = -abs(nums[v-1])
    # Phase 3: find first unmarked
    for i in range(n):
        if nums[i] > 0:
            return i + 1
    return n + 1
```

## Approach 2: Cycle Sort

<ApproachTag tag="interview" />

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Each element moved at most once" spaceNote="In-place" />

Place each number at its "correct" position: value v belongs at index v-1. Keep swapping until every element is either in its correct slot or out of range. Then scan for the first mismatch.

### Step-by-step Example

**Input:** `[3,4,-1,1]`

| i | nums[i] | correct idx | action | array |
| --- | --- | --- | --- | --- |
| 0 | 3 | 2 | swap(0,2) | [-1,4,3,1] |
| 0 | -1 | invalid | skip | [-1,4,3,1] |
| 1 | 4 | 3 | swap(1,3) | [-1,1,3,4] |
| 1 | 1 | 0 | swap(1,0) | [1,-1,3,4] |
| 1 | -1 | invalid | skip | [1,-1,3,4] |
| 2 | 3 | 2 ✓ | skip | [1,-1,3,4] |
| 3 | 4 | 3 ✓ | skip | [1,-1,3,4] |

**Scan:** index 0: nums[0]=1=1 ✓, index 1: nums[1]=-1≠2 → **return 2** ✓

### Pseudocode

```plaintext
function firstMissingPositive(nums):
    n = len(nums)
    i = 0
    while i < n:
        j = nums[i] - 1  # correct index for nums[i]
        if 1 <= nums[i] <= n and nums[j] != nums[i]:
            swap(nums[i], nums[j])
        else:
            i += 1
    for i in range(n):
        if nums[i] != i + 1:
            return i + 1
    return n + 1
```

<Aside type="caution" title="Common mistakes">
- **Index-as-hash — skipping Phase 1**: negatives in the array would cause `abs()` to produce indices pointing at wrong slots or out-of-range indices
- **Cycle sort — infinite loop on duplicates**: if `nums[i] == nums[nums[i]-1]`, swapping loops forever — always guard with `nums[j] != nums[i]` before swapping
- **Off-by-one**: value v belongs at index v-1, not v
</Aside>

## Interview Tips

<Aside type="tip" title="Key points to cover">
- State both constraints up front (O(n) time, O(1) space) — they together rule out sorting and hash sets
- The answer MUST be in [1, n+1] — use this bound to justify using the array as storage
- The index-as-hash approach is more commonly expected; cycle sort is a nice variant to mention
- For index-as-hash, emphasize that `abs()` in Phase 2 is necessary because earlier iterations may have already negated the value

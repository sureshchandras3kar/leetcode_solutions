## Problem Statement

Given an integer array `nums`, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, `j != k`, and `nums[i] + nums[j] + nums[k] == 0`. Notice that the solution set must not contain duplicate triplets.

<Aside type="tip" title="Key insights">
- "No duplicate triplets" is the hard part — sorting first makes duplicate-skipping easy.
- Fix one number (`nums[i]`), then reduce to Two Sum II for the rest of the array.
- The two-pointer approach here is the natural extension of Two Sum II to three numbers.
</Aside>

### Examples

| Input | Output |
| --- | --- |
| `[-1,0,1,2,-1,-4]` | `[[-1,-1,2],[-1,0,1]]` |
| `[0,1,1]` | `[]` |
| `[0,0,0]` | `[[0,0,0]]` |

## Constraints

- `3 <= nums.length <= 3000`
- `-10^5 <= nums[i] <= 10^5`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Financial reconciliation** — find three transactions that cancel each other out (sum to zero).
- **Chemical balancing** — find three reagents whose net charges sum to neutral.
- **3D vector cancellation** — find three force vectors that sum to zero.
- **Error detection** — find three data values whose sum reveals a specific error pattern.
</Aside>

## Approach 1: Sort + Two Pointers (Recommended)

<ApproachTag tag="recommended" />

Sort the array. For each index `i`, use two pointers — `left = i+1` and `right = n-1` — to find pairs that sum to `-nums[i]`. Skip duplicate values for `i` and for the inner pointers after finding a triplet.

<ComplexityBadges time="O(n²)" space="O(1)" timeColor="yellow" spaceColor="green" timeNote="Fixed outer + two-pointer inner" spaceNote="Sorting in-place" />

### Step-by-step Example

**Input:** `[-1, 0, 1, 2, -1, -4]`

Sorted: `[-4, -1, -1, 0, 1, 2]`

| i | nums[i] | left | right | sum | action |
| --- | --- | --- | --- | --- | --- |
| 0 | -4 | 1→4 | 5→5 | all < 0 | left advances to right |
| 1 | -1 | 2 | 5 | -1-1+2=0 | ✓ add [-1,-1,2], skip dups |
| 1 | -1 | 3 | 4 | -1+0+1=0 | ✓ add [-1,0,1] |
| 2 | -1 | skip | — | — | dup of i=1, continue |
| 3 | 0 | 4 | 5 | 0+1+2=3 | >0, right-- |
| 3 | 0 | left≥right | — | — | done |

### Pseudocode

```plaintext
function threeSum(nums):
    sort(nums)
    result = []
    for i in range(len(nums) - 2):
        if nums[i] > 0: break
        if i > 0 and nums[i] == nums[i-1]: continue  // skip dup outer
        left, right = i + 1, len(nums) - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                result.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left+1]: left++
                while left < right and nums[right] == nums[right-1]: right--
                left++; right--
            elif s < 0:
                left++
            else:
                right--
    return result
```

## Approach 2: Hash Set

<ApproachTag tag="simple" />

Sort the array. For each index `i`, scan `j` from `i+1` to `n-1` using a set to check whether `-(nums[i] + nums[j])` was already seen in the current inner scan. Because the array is sorted and we skip duplicate `i` values, sorted triplets are naturally deduplicated.

<ComplexityBadges time="O(n²)" space="O(n)" timeColor="yellow" spaceColor="yellow" timeNote="Outer × inner scan" spaceNote="Inner set per outer iteration" />

### Step-by-step Example

**Input:** `[-1, 0, 1, 2, -1, -4]` (after sorting: `[-4, -1, -1, 0, 1, 2]`)

For `i=1`, `nums[i]=-1`:
- `j=2`: need = -(-1 + -1) = 2, seen={}; 2 not in seen → add -1 to seen
- `j=3`: need = -(-1 + 0) = 1, seen={-1}; 1 not in seen → add 0 to seen
- `j=4`: need = -(-1 + 1) = 0, seen={-1,0}; 0 not in seen → add 1 to seen
- `j=5`: need = -(-1 + 2) = -1, seen={-1,0,1}; -1 **in seen** → add [-1,-1,2]

Wait — the triplet is `[nums[i], need, nums[j]]` = `[-1, -1, 2]` ✓

Continuing with `j=5` already recorded… next `i=3, nums[i]=0`:
- `j=4`: need = -(0+1) = -1, seen={}; not in seen → add 1
- `j=5`: need = -(0+2) = -2, seen={1}; not in seen → add 2

No new triplets at `i=3`.

### Pseudocode

```plaintext
function threeSumHashSet(nums):
    sort(nums)
    result = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]: continue
        seen = set()
        for j in range(i+1, len(nums)):
            need = -(nums[i] + nums[j])
            if need in seen:
                triplet = [nums[i], need, nums[j]]
                if triplet not in result:
                    result.append(triplet)
            seen.add(nums[j])
    return result
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Not skipping duplicates for the outer loop: `if i > 0 and nums[i] == nums[i-1]: continue`
- Not skipping duplicates for left/right pointers after finding a valid triplet
- Applying `if nums[i] > 0: break` too aggressively or not at all — it only works after sorting
- Forgetting to sort first — without sorting, neither duplicate-skipping nor early termination works
</Aside>

## Interview Tips

<Aside type="tip" title="Key points to discuss">
- **Frame it as "fix one + Two Sum II"** — interviewers love this framing because it shows systematic problem decomposition.
- **The duplicate-skipping logic is the trip wire** — walk through it carefully; most candidates forget one of the three skip conditions.
- **Time complexity is O(n²) and that is optimal** — no known algorithm does better for 3Sum.
- **The `if nums[i] > 0: break` optimization** — after sorting, if the smallest remaining number is positive, no triplet can sum to zero.
- **Follow-up — 4Sum (#18)?** Add one more outer loop and reduce to 3Sum. The pattern generalizes to k-Sum recursively.

## Problem Statement

Given a string containing digits from `2-9` inclusive, return all possible letter combinations that the number could represent on a traditional phone keypad.

A mapping of digit to letters (just like on the telephone buttons) is given below. Note that `1` does not map to any letters.

```
2 → "abc"
3 → "def"
4 → "ghi"
5 → "jkl"
6 → "mno"
7 → "pqrs"
8 → "tuv"
9 → "wxyz"
```

<Aside type="tip" title="What to notice first">
- The number of combinations grows exponentially: digits "23" produces 3 × 3 = 9 combinations.
- You must explore all possible paths through the digit-to-letter mapping.
- This is a classic exploration problem solved by backtracking or BFS.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| `"23"` | `["ad", "ae", "af", "bd", "be", "bf", "cd", "ce", "cf"]` | 2 maps to "abc", 3 maps to "def"; all pairings |
| `""` | `[]` | Empty input yields empty result |
| `"2"` | `["a", "b", "c"]` | Single digit maps directly to its letters |
| `"234"` | 27 combinations | 3 × 3 × 3 = 27 total combinations |

## Constraints

- `0 <= digits.length <= 4`
- `digits[i]` is a digit in the range `['2', '9']`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Predictive text / autocomplete** — phone keyboards and T9 input systems generate word suggestions from digit sequences.
- **Password generation patterns** — exploring all possible character sequences for brute-force testing or policy compliance validation.
- **Telephony routing** — mapping numeric sequences to multiple possible extension names or office destinations.
- **Chess move enumeration** — generating all legal move combinations from a numeric board position encoding.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Returns | Best When |
| --- | --- | --- | --- | --- |
| Backtracking (Recursive) | O(4^n) | O(4^n) | All combinations | Want clear recursive structure, understanding decision tree |

- Pick **Backtracking** if you are learning recursion and decision trees.
- Pick **BFS** if you prefer iterative solutions or want to see level-by-level growth.

## Approach 1: Backtracking (Recommended)

<ApproachTag tag="recommended" />

Build combinations by recursively appending letters. For each digit, try all letters it maps to, then recurse on the next digit. When all digits are processed, save the combination.

The key insight is that we make a *choice* at each digit (which letter to use) and then recursively solve the subproblem for the remaining digits.

<ComplexityBadges time="O(4^n)" space="O(4^n)" timeColor="red" spaceColor="red" timeNote="4^n combinations" spaceNote="Result storage" />

### Step-by-step Example

**Input:** `digits = "23"`

| Depth | Digit | Current | Action |
| --- | --- | --- | --- |
| 0 | "2" | "" | Start backtracking |
| 1 | "3" | "a" | Process 2 → "a", now explore 3 |
| 2 | - | "ad" | Process 3 → "d", save "ad" ✓ |
| 2 | - | "ae" | Process 3 → "e", save "ae" ✓ |
| 2 | - | "af" | Process 3 → "f", save "af" ✓ |
| 1 | "3" | "b" | Backtrack to 2 → "b", now explore 3 |
| 2 | - | "bd" | Process 3 → "d", save "bd" ✓ |
| ... | ... | ... | Continue for "c" + all letters in "def" |

### Pseudocode

```plaintext
function letterCombinationsBacktracking(digits):
    if digits is empty:
        return []
    
    phoneMap = {
        "2": "abc", "3": "def", "4": "ghi", "5": "jkl",
        "6": "mno", "7": "pqrs", "8": "tuv", "9": "wxyz"
    }
    
    result = []
    
    function backtrack(index, current):
        if index == len(digits):
            result.append(current)
            return
        
        digit = digits[index]
        letters = phoneMap[digit]
        for letter in letters:
            backtrack(index + 1, current + letter)
    
    backtrack(0, "")
    return result
```

## Approach 2: BFS (Level-Order Iteration)

<ApproachTag tag="interview" />

Build combinations iteratively, processing one digit at a time. Start with a queue containing an empty string. For each digit, create new combinations by appending each of its letters to all current combinations.

This approach grows the result level by level, making it intuitive for those who prefer iterative over recursive solutions.

<ComplexityBadges time="O(4^n)" space="O(4^n)" timeColor="red" spaceColor="red" timeNote="4^n combinations" spaceColor="4^n queue size" />

### Step-by-step Example

**Input:** `digits = "23"`

| Digit | Queue Before | Queue After | Action |
| --- | --- | --- | --- |
| "2" | `[""]` | `["a", "b", "c"]` | Append each of "abc" to "" |
| "3" | `["a", "b", "c"]` | `["ad", "ae", "af", "bd", "be", "bf", "cd", "ce", "cf"]` | Append each of "def" to each in queue |

### Pseudocode

```plaintext
function letterCombinationsBFS(digits):
    if digits is empty:
        return []
    
    phoneMap = {
        "2": "abc", "3": "def", "4": "ghi", "5": "jkl",
        "6": "mno", "7": "pqrs", "8": "tuv", "9": "wxyz"
    }
    
    queue = [""]
    
    for digit in digits:
        newQueue = []
        letters = phoneMap[digit]
        for current in queue:
            for letter in letters:
                newQueue.append(current + letter)
        queue = newQueue
    
    return queue
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to handle the empty string case (return `[]`, not `[""]`).
- Using a static phone map incorrectly (ensure digit '0' and '1' are not included or handled separately).
- In the BFS approach, clearing the queue without saving it first (always save results before processing the next digit).
- Off-by-one errors in the backtracking base case (ensure index == len(digits), not index > len(digits)).
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Backtracking vs BFS:** Both are O(4^n) time and space. Backtracking is more natural for recursive explorers; BFS is better if you want to demonstrate understanding of level-order growth.
- **Edge cases:** Emphasize that empty input should return an empty list, not a list with one empty string.
- **Phone keypad knowledge:** Be ready to explain why digits 0 and 1 are not included (historically, they had no letters on phone keypads).
- **Follow-up — K-digit combinations:** If asked to generate only k-length combinations, add a parameter and check `len(current) == k` before saving.
</Aside>

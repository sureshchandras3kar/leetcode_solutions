## Problem Statement

Given the `head` of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the `next` pointer. Internally, `pos` is used to denote the index of the node that tail's `next` pointer is connected to. Note that `pos` is **not passed as a parameter**.

Return `true` if there is a cycle in the linked list. Otherwise, return `false`.

<Aside type="tip" title="What to notice first">
- A cycle exists when a node's `next` pointer eventually points back to a node we've already visited.
- **Floyd's Algorithm** uses two pointers at different speeds: if they ever meet, a cycle exists. This works because in a circular list, the fast pointer will eventually "lap" the slow pointer.
- The **Hash Set** approach tracks visited nodes; if we encounter a node twice, a cycle exists.
- Both approaches have the same O(n) time complexity, but Floyd's saves space with O(1) vs. O(n).
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| Head: `[3,2,0,-4]`, pos: `1` | `true` | Cycle: 2 → 0 → -4 → 2 |
| Head: `[1,2]`, pos: `0` | `true` | Cycle: 2 → 1 → 2 |
| Head: `[1]`, pos: `-1` | `false` | No cycle |

## Constraints

- The number of the nodes in the list is in the range `[0, 10^4]`.
- `-10^5 <= Node.val <= 10^5`
- `pos` is `-1` or a **valid index** in the linked-list.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Garbage collection** — detect circular references in memory to prevent infinite loops during cleanup.
- **Graph traversal** — identify cycles in directed graphs to ensure algorithms terminate.
- **Data structure validation** — verify linked lists are well-formed (no accidental cycles).
- **Deadlock detection** — in concurrent systems, identify circular wait conditions between threads or resources.
- **Infinite loop prevention** — detect cycles in dependency chains (e.g., in build systems or package managers).
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Floyd's Algorithm | O(n) | O(1) | Space is limited; elegant two-pointer technique |
| Hash Set | O(n) | O(n) | Space is available; easier to understand |

## Which approach should you learn first?

- Learn **Floyd's Algorithm** first — it is the **optimal** solution and a classic interview technique.
- Use **Hash Set** as a fallback if you forget the two-pointer logic; it is straightforward and correct.

## Approach 1: Floyd's Algorithm (Tortoise & Hare)

<ApproachTag tag="recommended" />

Use two pointers starting at the head: a slow pointer that advances one step at a time, and a fast pointer that advances two steps at a time. If a cycle exists, the fast pointer will eventually catch up to (meet) the slow pointer. If no cycle exists, the fast pointer will reach the end (null).

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass with two pointers" spaceNote="Only two pointer variables" />

### Step-by-step Example

**Input:** `[3, 2, 0, -4]` with cycle at index 1

| Step | Slow Position | Fast Position | Notes |
| --- | --- | --- | --- |
| Start | 3 | 3 | Both start at head |
| Step 1 | 2 | 0 | Slow moves 1, fast moves 2 |
| Step 2 | 0 | 2 | Slow moves 1, fast moves 2 |
| Step 3 | -4 | -4 | Slow moves 1, fast moves 2 |
| Step 4 | 2 | 0 | Both in cycle, continue |
| Step 5 | 0 | -4 | Fast closing in |
| Step 6 | -4 | 2 | Still moving |
| Step 7 | 2 | 0 | Pointers converge |
| Step 8 | 0 | -4 | **Slow == Fast** → Cycle detected! |

### Pseudocode

```plaintext
function hasCycle(head):
    if head is null or head.next is null:
        return false
    
    slow = head
    fast = head
    
    while fast is not null and fast.next is not null:
        slow = slow.next                    // Move 1 step
        fast = fast.next.next              // Move 2 steps
        
        if slow == fast:
            return true                     // Collision detected
    
    return false                            // No cycle found
```

## Approach 2: Hash Set

<ApproachTag tag="simple" />

Iterate through the linked list, storing each visited node in a set. If we encounter a node that is already in the set, a cycle exists. If we reach the end of the list (null), there is no cycle.

<ComplexityBadges time="O(n)" space="O(n)" timeColor="green" spaceColor="yellow" timeNote="Single pass through list" spaceNote="Set grows with visited nodes" />

### Step-by-step Example

**Input:** `[3, 2, 0, -4]` with cycle at index 1

| Node | In Set? | Action |
| --- | --- | --- |
| 3 | No | Add to set → `{3}` |
| 2 | No | Add to set → `{3, 2}` |
| 0 | No | Add to set → `{3, 2, 0}` |
| -4 | No | Add to set → `{3, 2, 0, -4}` |
| 2 | **Yes** | **Cycle detected!** Return `true` |

### Pseudocode

```plaintext
function hasCycle(head):
    seen = empty set
    current = head
    
    while current is not null:
        if current is in seen:
            return true                     // Found a cycle
        seen.add(current)
        current = current.next
    
    return false                            // No cycle found
```

<Aside type="caution" title="Watch for these pitfalls">
- **Null pointer checks:** Always check if `head` is null or `head.next` is null before entering the loop in Floyd's algorithm.
- **Fast pointer safety:** In Floyd's algorithm, check that `fast` and `fast.next` are not null before advancing; otherwise, you risk a null pointer exception.
- **Hash set memory:** The hash set approach uses O(n) extra space, which can be problematic for very large lists.
- **Pointer identity vs. value:** Use pointer comparison (`==`), not value comparison, to detect cycles. Two different nodes with the same value are not the same node.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why Floyd's Algorithm?** It is O(1) space and demonstrates sophisticated two-pointer technique. This is the expected solution for optimal interviews.
- **Hash Set alternative:** If you struggle to recall Floyd's algorithm, the hash set approach is correct and O(n) space is often acceptable.
- **Proof of correctness:** Floyd's algorithm works because in a circular list, the difference in speeds guarantees eventual collision. The relative speed of 2 vs. 1 means the fast pointer "laps" the slow pointer.
- **Edge cases:** Empty list, single node, two-node cycle, and late-cycle entry all test your implementation thoroughly.
- **Follow-up:** Problem #142 asks for the cycle start node — Floyd's algorithm can be extended to find it in two phases.

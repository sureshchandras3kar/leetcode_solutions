## Problem Statement

Given the `root` of a Binary Search Tree (BST), return the minimum absolute difference between the values of any two different nodes in the tree.

<Aside type="tip" title="What to notice first">
- In a BST, an **in-order traversal yields values in ascending sorted order**.
- The minimum difference must occur between **consecutive values** in sorted order.
- You do not need to compare all pairs; sorting the values is implicit via in-order traversal.
</Aside>

### Examples

| Input | Output | Explanation |
| --- | --- | --- |
| Tree: `4, 2→1, 2→3, 6` | `1` | Sorted values: `[1, 2, 3, 4, 6]`. Min diff = `2 - 1 = 1` |
| Tree: `1, 0, 48→24, 48→null` | `24` | Sorted values: `[0, 1, 24, 48]`. Min diff = `24 - 0 = 24` |

## Constraints

- The number of nodes in the tree is in the range `[2, 10^4]`.
- `0 <= Node.val <= 10^5`
- All the values of the tree are **unique**.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Sensor calibration** — given multiple sensor readings, find the two closest readings to identify if sensors are drifting.
- **Stock price pairs** — find the two stock prices with the smallest difference to identify arbitrage opportunities.
- **Machine tolerance ranges** — verify that all manufactured parts fall within acceptable variance.
- **Binary classification thresholds** — determine the optimal boundary between two classes by minimizing class difference.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Key Idea | Best When |
| --- | --- | --- | --- | --- |
| In-Order DFS | O(n) | O(h) | Visit nodes in sorted order; compare adjacent pairs only | **General case** — standard BST usage |
| BFS Level-Order | O(n log n) | O(w) | Collect all values, sort, then find min diff | Tree structure unknown or comparison required |

## Which approach should you learn first?

- Pick **In-Order DFS** if you want the most elegant and efficient solution.
- Pick **BFS Level-Order** if you prefer a simpler mental model (collect all, sort, compare).

## Approach 1: In-Order DFS (Recommended)

<ApproachTag tag="recommended" />

Traverse the BST in in-order (left → node → right), which visits nodes in ascending sorted order. As you visit each node, compare its value with the previously visited value. The minimum difference must occur between two consecutive values in this sorted sequence.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceNote="Recursion stack" />

### Step-by-step Example

**Input:** Tree with values `[4, 2, 6, 1, 3]`

| Step | Node | prev | Diff | min_diff | Action |
| --- | --- | --- | --- | --- | --- |
| 1 | 1 | null | — | ∞ | First node, store prev = 1 |
| 2 | 2 | 1 | 2 - 1 = 1 | 1 | Update min_diff = 1 |
| 3 | 3 | 2 | 3 - 2 = 1 | 1 | min_diff = 1 |
| 4 | 4 | 3 | 4 - 3 = 1 | 1 | min_diff = 1 |
| 5 | 6 | 4 | 6 - 4 = 2 | 1 | min_diff stays 1 |

### Pseudocode

```plaintext
function getMinimumDifference_inorder(root):
    min_diff = infinity
    prev = null

    function inorder(node):
        if node is null:
            return

        inorder(node.left)

        // Visit node
        if prev is not null:
            min_diff = min(min_diff, node.val - prev)
        prev = node.val

        inorder(node.right)

    inorder(root)
    return min_diff
```

## Approach 2: BFS Level-Order

<ApproachTag tag="interview" />

Perform a level-order (breadth-first) traversal to collect all node values, sort them, then iterate through consecutive pairs to find the minimum difference.

This approach trades O(n log n) time for a simpler, more straightforward algorithm that does not rely on understanding BST properties.

<ComplexityBadges time="O(n log n)" space="O(w)" timeColor="yellow" spaceColor="yellow" timeNote="Sorting overhead" spaceNote="Queue + values" />

### Step-by-step Example

**Input:** Tree with values `[4, 2, 6, 1, 3]`

1. **BFS Traversal:** Collect `[4, 2, 6, 1, 3]`
2. **Sort:** `[1, 2, 3, 4, 6]`
3. **Find min diff between consecutive pairs:**
   - `2 - 1 = 1`
   - `3 - 2 = 1`
   - `4 - 3 = 1`
   - `6 - 4 = 2`
   - **Result:** `1`

### Pseudocode

```plaintext
function getMinimumDifference_bfs(root):
    if root is null:
        return 0

    values = []
    queue = [root]

    // Collect all values via BFS
    while queue is not empty:
        node = queue.pop()
        values.append(node.val)
        if node.left:
            queue.push(node.left)
        if node.right:
            queue.push(node.right)

    // Sort values
    sort(values)

    // Find min diff between consecutive values
    min_diff = infinity
    for i in range(1, len(values)):
        min_diff = min(min_diff, values[i] - values[i-1])

    return min_diff
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Comparing all pairs:** Forgetting that minimum diff must occur between consecutive sorted values. O(n²) comparisons are unnecessary.
- **Ignoring BST property:** The BST property guarantees in-order traversal is sorted. Use it!
- **Off-by-one errors:** When iterating through sorted values, ensure you compare `values[i] - values[i-1]` for all i ≥ 1.
- **Not handling single-node trees:** Although constraints say n ≥ 2, defensive checks are safe.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **In-Order DFS vs BFS:** In-order naturally maintains sorted order, eliminating the need for explicit sorting. This is the signature BST property and the interviewer likely wants to see you leverage it.
- **Space complexity:** In-Order uses O(h) recursion stack (often optimal for balanced trees). BFS uses O(w) for the queue, which can be O(n) for skewed trees or wide trees.
- **Edge cases:** Empty tree (handled by constraints), single edge difference, duplicate values (problem states all unique).
- **Follow-up:** "Can you do this iteratively?" Yes — use an explicit stack to simulate the recursion for in-order traversal.
</Aside>

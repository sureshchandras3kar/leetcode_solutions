## Problem Statement

Given the `root` of a binary search tree and an integer `k`, return the **kth smallest value** (1-indexed) of all the values of the nodes in the tree.

<Aside type="tip" title="What to notice first">
- In a BST, **in-order traversal visits nodes in ascending sorted order**.
- You can count as you traverse and stop at the kth node.
- No need to store all values; process on-the-fly.
</Aside>

### Examples

| Input | k | Output | Explanation |
| --- | --- | --- | --- |
| Tree: `3, 1→null, 1→2, 4` | 1 | 1 | Sorted: `[1, 2, 3, 4]`, 1st = 1 |
| Tree: `3, 1→null, 1→2, 4` | 3 | 2 | Sorted: `[1, 2, 3, 4]`, 3rd = 2 |
| Tree: `5, 3, 6` | 1 | 3 | Sorted: `[3, 5, 6]`, 1st = 3 |

## Constraints

- The number of nodes in the tree is `n`.
- `1 <= k <= n <= 10^4`
- `0 <= Node.val <= 10^4`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Percentile ranking** — find the kth percentile student score in a sorted dataset.
- **Leaderboard systems** — retrieve the kth highest score efficiently.
- **Data streaming** — find the kth smallest event timestamp without storing all events.
- **Statistical analysis** — extract medians and quartiles from sorted data.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Key Idea | Best When |
| --- | --- | --- | --- | --- |
| In-Order DFS | O(n) | O(h) | Recursive in-order, count k nodes, return | **General case** — simple and elegant |
| Morris In-Order | O(n) | O(1) | Thread-based traversal, no recursion stack | **Space-critical** — embedded systems, interview edge case |

## Which approach should you learn first?

- Pick **In-Order DFS** as your primary solution.
- Learn **Morris In-Order** if you want to optimize space to O(1).

## Approach 1: In-Order DFS (Recommended)

<ApproachTag tag="recommended" />

Recursively traverse the tree in in-order (left → node → right). Maintain a counter that increments with each node visited. When the counter equals k, return the current node's value.

<ComplexityBadges time="O(n)" space="O(h)" timeColor="green" spaceColor="green" timeNote="Worst case: visit all nodes" spaceColor="Recursion depth" />

### Step-by-step Example

**Input:** Tree with values `[3, 1, 4, 2]`, `k = 1`

In-order sequence: `[1, 2, 3, 4]`

| Step | Node | count | Action |
| --- | --- | --- | --- |
| 1 | 1 | 1 | count == k, return 1 ✓ |

### Pseudocode

```plaintext
function kthSmallest_inorder(root, k):
    count = 0
    result = null

    function inorder(node):
        if node is null or result is not null:
            return

        inorder(node.left)

        count++
        if count == k:
            result = node.val
            return

        inorder(node.right)

    inorder(root)
    return result
```

## Approach 2: Morris In-Order Traversal

<ApproachTag tag="advanced" />

Morris traversal enables in-order traversal without recursion or an explicit stack. It uses "threading" — temporary pointers to connect nodes. This achieves O(1) space.

The algorithm works in two passes per node: on the first visit, create a thread back to the current node; on the second visit, remove the thread and process the node.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="All nodes visited" spaceNote="No extra structure" />

### Step-by-step Example

**Input:** Tree with values `[3, 1, 4, 2]`, `k = 1`

1. Start at 3 (root).
2. Find predecessor of 3: it's 2.
3. Thread 2 → 3 (first visit to 2).
4. Move to left child 1.
5. 1 has no left child. Visit 1, count = 1. **Return 1**.

### Pseudocode

```plaintext
function kthSmallest_morris(root, k):
    count = 0
    current = root

    while current is not null:
        if current.left is null:
            // No left child: visit current node
            count++
            if count == k:
                return current.val
            current = current.right
        else:
            // Find in-order predecessor
            predecessor = current.left
            while predecessor.right and predecessor.right != current:
                predecessor = predecessor.right

            if predecessor.right is null:
                // First visit: create thread
                predecessor.right = current
                current = current.left
            else:
                // Second visit: remove thread, process current
                predecessor.right = null
                count++
                if count == k:
                    return current.val
                current = current.right

    return -1
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Counting from 0 vs 1:** The problem uses 1-indexed k. Ensure your counter starts at 0 and increments *before* checking k.
- **Early termination:** Once the kth element is found, avoid further recursion. Use a flag or check `result != null`.
- **Off-by-one errors in Morris:** Ensure threads are created and removed correctly during first and second visits.
- **Modifying the tree (Morris):** Morris traversal temporarily modifies the tree. Ensure threads are restored for re-entrant code.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **DFS vs Morris:** DFS is simpler and more intuitive. Morris is an advanced technique that shows deep understanding of tree traversal.
- **Space complexity:** In-Order DFS uses O(h) recursion stack. For balanced trees, h = log(n). For skewed trees, h = n.
- **Follow-up:** "Can you solve this in O(log n) time if the BST is augmented with subtree sizes?" Yes — use binary search within the tree structure.
- **Edge cases:** k = 1 (smallest element), k = n (largest element), single-node trees.
</Aside>

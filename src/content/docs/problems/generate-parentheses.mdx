## Problem Statement

Given `n` pairs of parentheses, write a function to **generate all combinations of well-formed parentheses**.

<Aside type="tip" title="What to notice first">
- A valid parenthesis string has equal numbers of `(` and `)`.
- At any point while building, open count must be >= close count.
- You can only add `(` if open < n.
- You can only add `)` if close < open.
</Aside>

### Examples

| n | Output | Count |
| --- | --- | --- |
| 1 | `["()"]` | 1 |
| 2 | `["(())","()()"]` | 2 |
| 3 | `["((()))","(()())","(())()","()((})","()()()"]` | 5 |

## Constraints

- `1 <= n <= 8`
- The number of well-formed strings grows as the nth Catalan number: C(n) = (2n)! / ((n+1)! * n!)

## Complexity Comparison

| Approach | Time | Space | Notes |
| --- | --- | --- | --- |
| Backtracking | O(4^n / √n) | O(n) | Prunes invalid paths early; intuitive recursion |
| Dynamic Programming | O(4^n / √n) | O(n) | Builds bottom-up; combines subproblems |

Both approaches have the same asymptotic complexity (Catalan number), but backtracking prunes earlier.

## Approach 1: Backtracking (Recommended)

<ApproachTag tag="recommended" />

Use backtracking to generate parentheses recursively. At each step, decide whether to add an opening or closing parenthesis:
- Add `(` if we have fewer than n opening parentheses.
- Add `)` if we have fewer closing than opening parentheses (maintains validity).

This prunes invalid paths immediately, making it more efficient than brute force.

<ComplexityBadges time="O(4^n / √n)" space="O(n)" timeColor="yellow" spaceColor="green" timeNote="Catalan number growth" spaceNote="Recursion depth + result" />

### Step-by-step Example

**Input:** `n = 2`

```
Start: ""
├─ Add '(': "("
│  ├─ Add '(': "(("
│  │  └─ Add ')': "(()"
│  │     └─ Add ')': "(())"  ✓
│  └─ Add ')': "()"
│     └─ Add '(': "()("
│        └─ Add ')': "()()"  ✓
└─ Can't add ')' yet (left == 0)

Result: ["(())", "()()"]
```

### Pseudocode

```plaintext
function generateParenthesis(n):
    result = []
    
    def backtrack(current, leftCount, rightCount):
        if len(current) == 2 * n:
            result.append(current)
            return
        
        if leftCount < n:
            backtrack(current + "(", leftCount + 1, rightCount)
        
        if rightCount < leftCount:
            backtrack(current + ")", leftCount, rightCount + 1)
    
    backtrack("", 0, 0)
    return result
```

## Approach 2: Dynamic Programming

<ApproachTag tag="simple" />

Build valid parentheses combinations bottom-up. For i pairs, insert all valid combinations of j pairs inside parentheses, then append all valid combinations of remaining (i-1-j) pairs.

The recurrence: `DP[i]` = `"(" + DP[j] + ")" + DP[i-1-j]` for all valid j.

<ComplexityBadges time="O(4^n / √n)" space="O(n)" timeColor="yellow" spaceColor="green" timeNote="Catalan number" spaceNote="DP table + recursion" />

### Step-by-step Example

**Input:** `n = 2`

```
DP[0] = [""]
DP[1] = ["()"]        (from: "(" + DP[0] + ")" + DP[0])
DP[2] = [
  "(" + "()" + ")" + "" = "(())",
  "(" + "" + ")" + "()" = "()()"
]
```

### Pseudocode

```plaintext
function generateParenthesis(n):
    dp = [[] for _ in range(n + 1)]
    dp[0] = [""]
    
    for i in range(1, n + 1):
        for j in range(i):
            for left in dp[j]:
                for right in dp[i - 1 - j]:
                    dp[i].append("(" + left + ")" + right)
    
    return dp[n]
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Forgetting the closing parenthesis condition:** You must check `rightCount < leftCount` to avoid closing before opening.
- **Base case logic:** Ensure `len(current) == 2 * n` is the termination condition.
- **DP indexing:** Ensure loop bounds and DP table initialization are correct (off-by-one errors).
- **String building:** In languages like Java, use StringBuilder for efficiency to avoid O(n) concatenation overhead.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Backtracking vs. DP:** Backtracking is more intuitive and prunes early. DP is more algorithmic and bottom-up.
- **Why Catalan numbers?** The nth Catalan number counts all valid parenthesis combinations; knowing this helps validate your result count.
- **Follow-up:** Can you count the solutions without generating them? (Yes—just return DP[n].length or count during recursion.)
- **Space optimization:** Neither approach reduces space below O(n) because the output itself is exponential.
</Aside>

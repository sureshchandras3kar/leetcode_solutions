## Problem Statement

Given two integer arrays `nums1` and `nums2`, return an array of their **intersection**. Each element in the result must appear only **once**, and the result can be in any order.

<Aside type="tip" title="What to notice first">
- The result requires **unique** values — even if a number appears multiple times in both arrays, it appears only once in the output.
- The hash set approach is fastest when the arrays are unsorted or of different sizes.
- The two-pointer approach shines when the input is already sorted or when no extra memory is available.
</Aside>

### Examples

| nums1 | nums2 | Output | Explanation |
| --- | --- | --- | --- |
| `[1, 2, 2, 1]` | `[2, 2]` | `[2]` | `2` is common; duplicates collapsed |
| `[4, 9, 5]` | `[9, 4, 9, 8, 4]` | `[9, 4]` | `9` and `4` are common |

## Constraints

- `1 <= nums1.length, nums2.length <= 1000`
- `0 <= nums1[i], nums2[i] <= 1000`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Friend suggestions** — find users that two people both follow.
- **Tag matching** — find categories that two documents share.
- **Inventory cross-check** — find product SKUs that appear in both a purchase order and a warehouse manifest.
- **DNS cache deduplication** — find hostnames resolved by both of two DNS resolvers over a time window.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Best When |
| --- | --- | --- | --- |
| Hash Set | O(n + m) | O(min(n, m)) | Unsorted arrays; fast lookup |
| Two Pointers | O(n log n + m log m) | O(1)\* | Arrays are already sorted or memory is scarce |

\* O(1) extra space beyond the output array (sort is in-place).

## Which approach should you learn first?

- Learn **Hash Set** first — it is the faster approach for unsorted inputs and the standard expectation in interviews.
- Learn **Two Pointers** as a follow-up — it demonstrates the general sorted-merge pattern used across many problems.

## Approach 1: Hash Set

<ApproachTag tag="recommended" />

Convert `nums1` into a set to deduplicate it. Scan `nums2` and collect every element that is in the set into a result set (to prevent duplicates in the output). Convert the result set to a list.

<ComplexityBadges time="O(n + m)" space="O(min(n, m))" timeColor="green" spaceColor="yellow" timeNote="One pass each" spaceNote="Set of the smaller array" />

### Step-by-step Example

**Input:** `nums1 = [4, 9, 5]`, `nums2 = [9, 4, 9, 8, 4]`

| Phase | Action | Result set |
| --- | --- | --- |
| Build set from nums1 | `{4, 9, 5}` | — |
| Check 9 from nums2 | 9 ∈ set1 | `{9}` |
| Check 4 from nums2 | 4 ∈ set1 | `{9, 4}` |
| Check 9 again | Already in result | `{9, 4}` |
| Check 8 | 8 ∉ set1 | `{9, 4}` |
| Check 4 again | Already in result | `{9, 4}` |
| **Return** | | `[9, 4]` |

### Pseudocode

```plaintext
function intersection(nums1, nums2):
    set1 = set(nums1)
    result = set()
    for num in nums2:
        if num in set1:
            result.add(num)
    return list(result)
```

## Approach 2: Two Pointers

<ApproachTag tag="interview" />

Sort both arrays. Use two pointers starting at the beginning of each. Advance the pointer pointing to the smaller value. When both pointers point to equal values, record the result (skipping consecutive duplicates) and advance both.

<ComplexityBadges time="O(n log n + m log m)" space="O(1)" timeColor="yellow" spaceColor="green" timeNote="Sort dominates" spaceNote="In-place sort, no extra structure" />

### Step-by-step Example

**Input:** `nums1 = [4, 9, 5]`, `nums2 = [9, 4, 9, 8, 4]`

After sorting: `nums1 = [4, 5, 9]`, `nums2 = [4, 4, 8, 9, 9]`

| i | j | nums1[i] | nums2[j] | Action | result |
| --- | --- | --- | --- | --- | --- |
| 0 | 0 | 4 | 4 | Equal → add, both++ | `[4]` |
| 1 | 2 | 5 | 8 | 5 < 8 → i++ | `[4]` |
| 2 | 2 | 9 | 8 | 9 > 8 → j++ | `[4]` |
| 2 | 3 | 9 | 9 | Equal → add, both++ | `[4, 9]` |
| 3 | 5 | — | — | Done | `[4, 9]` |

### Pseudocode

```plaintext
function intersection(nums1, nums2):
    sort(nums1); sort(nums2)
    result = []
    i, j = 0, 0
    while i < len(nums1) and j < len(nums2):
        if nums1[i] == nums2[j]:
            if not result or result[-1] != nums1[i]:
                result.append(nums1[i])
            i++; j++
        elif nums1[i] < nums2[j]:
            i++
        else:
            j++
    return result
```

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to deduplicate: if the same value appears in both arrays multiple times, both approaches must ensure it appears only once in the output — the hash set does this automatically, but the two-pointer approach needs an explicit skip.
- Sorting modifies the input arrays — if the caller needs the original order, sort copies instead.
- Confusing this problem (#349, unique intersection) with #350 (Intersection of Two Arrays II), which allows duplicates in the result.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Hash set vs two pointers:** hash set wins on speed (O(n+m)), two pointers wins on space (O(1) extra). Know when to use each.
- **Follow-up — if one array is very large and the other is small?** Build the set from the smaller array to minimise memory.
- **Follow-up — if the arrays are already sorted?** Skip the sort cost and go directly to the two-pointer scan.
- **Follow-up — intersection with duplicates (#350)?** Replace the result set with a frequency map and take the minimum count of each common element.
</Aside>

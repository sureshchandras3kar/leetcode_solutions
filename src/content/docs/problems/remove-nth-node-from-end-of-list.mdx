## Problem Statement

Given the `head` of a linked list, remove the `nth` node from the end of the list and return the head of the modified list.

You must solve this **in one pass** when possible, but a two-pass solution is a good warm-up.

<Aside type="tip" title="What to notice first">
- The key challenge is identifying the node *before* the one you want to remove, not the node itself.
- Removing the head is a special case — the result is `head.next`.
- Using a dummy node elegantly handles the head removal case without branching.
- The "one pass" constraint forces you to maintain a gap between two pointers.
</Aside>

### Examples

| Input | n | Output | Explanation |
| --- | --- | --- | --- |
| `1 → 2 → 3 → 4 → 5` | 2 | `1 → 2 → 3 → 5` | The 4th node (value 4) is removed. |
| `1` | 1 | (empty) | Single node is removed; result is empty. |
| `1 → 2` | 1 | `1` | Remove the 2nd node (end); result is `[1]`. |
| `1 → 2` | 2 | `2` | Remove the 1st node (head); result is `[2]`. |

## Constraints

- The number of nodes in the list is `sz`.
- `1 <= sz <= 30`
- `0 < n <= sz`
- All node values are unique.

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Circular buffer eviction** — remove the nth most recent element to implement an LRU cache.
- **Audio playback** — skip the nth song from the end of a playlist queue.
- **Undo/redo stacks** — remove the nth action from the end of the history when memory is limited.
- **Log rotation** — discard the nth oldest log entry when the buffer is full.
- **Packet processing** — drop the nth packet from the back of a transmission queue.
</Aside>

## Approach 1: Two Pass

<ApproachTag tag="simple" />

Count the total number of nodes in the first pass, then find and skip the target node in the second pass.

This approach is straightforward: first measure the list, then navigate to the node before the one to remove.

<ComplexityBadges time="O(L)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Two passes" spaceColor="No extra memory" />

### Step-by-step Example

**Input:** `1 → 2 → 3 → 4 → 5`, `n = 2`

**Pass 1 (Count):** Length = 5

**Pass 2 (Remove):** We want to skip the node at position `5 - 2 = 3` from the start (0-indexed: position 3).
- Navigate to position 2 (the node with value 3).
- Set `node.next = node.next.next` (skip the 4).
- Result: `1 → 2 → 3 → 5`

**Another case:** `n = 1` (remove the last node)

**Pass 1 (Count):** Length = 5

**Pass 2 (Remove):** We want to skip the node at position `5 - 1 = 4` from the start.
- Navigate to position 3 (the node with value 4).
- Set `node.next = node.next.next` (skip the 5).
- Result: `1 → 2 → 3 → 4`

**Edge case:** `1`, `n = 1` (remove the only node)

**Pass 1 (Count):** Length = 1

Since `length == n`, return `head.next`, which is `null`.
- Result: (empty list)

### Pseudocode

```plaintext
function removeNthNodeTwoPass(head, n):
    // First pass: count total nodes
    length = 0
    current = head
    while current != null:
        length++
        current = current.next

    // Edge case: removing the head
    if length == n:
        return head.next

    // Second pass: find the node before the one to remove
    current = head
    for i from 0 to length - n - 2:
        current = current.next

    // Remove the nth node
    current.next = current.next.next
    return head
```

## Approach 2: Two Pointers (One Pass - Recommended)

<ApproachTag tag="recommended" />

Use a dummy node and two pointers (fast and slow) with a controlled gap. Move both pointers in lockstep; when the fast pointer reaches the end, the slow pointer will be at the node *before* the one to remove.

The dummy node is the key: it cleanly handles the edge case of removing the head without special branching.

<ComplexityBadges time="O(L)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceColor="No extra memory" />

### Step-by-step Example

**Input:** `1 → 2 → 3 → 4 → 5`, `n = 2`

**Setup:** Create dummy: `0 → 1 → 2 → 3 → 4 → 5`
- Slow = dummy, fast = dummy

**Move fast pointer `n+1 = 3` steps ahead:**
- Step 1: fast = `1`
- Step 2: fast = `2`
- Step 3: fast = `3`

Now there is a gap of 2 nodes between slow (dummy) and fast (3).

**Move both pointers until fast reaches the end:**
- Move 1: slow = `1`, fast = `4`
- Move 2: slow = `2`, fast = `5`
- Move 3: slow = `3`, fast = `null`

Fast is now `null`. Slow is at node `3`.

**Remove the next node:** `slow.next = slow.next.next`
- `3.next = 5` (skip 4)

**Result:** `0 → 1 → 2 → 3 → 5`, return `dummy.next` = `1 → 2 → 3 → 5`

**Edge case:** `1 → 2`, `n = 2` (remove the head)

**Setup:** Create dummy: `0 → 1 → 2`
- Slow = dummy, fast = dummy

**Move fast pointer `n+1 = 3` steps ahead:**
- Step 1: fast = `1`
- Step 2: fast = `2`
- Step 3: fast = `null`

**Move both pointers until fast reaches the end:**
- fast is already null, so the loop does not run.

**Remove the next node:** `slow.next = slow.next.next`
- dummy.next = 2 (skip 1)

**Result:** `0 → 2`, return `dummy.next` = `2`

### Pseudocode

```plaintext
function removeNthNodeTwoPointers(head, n):
    dummy = new Node(0)
    dummy.next = head
    slow = dummy
    fast = dummy

    // Move fast pointer n+1 steps ahead
    for i from 0 to n:
        if fast == null:
            return head
        fast = fast.next

    // Move both pointers until fast reaches the end
    while fast != null:
        slow = slow.next
        fast = fast.next

    // Remove the nth node
    slow.next = slow.next.next
    return dummy.next
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- **Not handling the head removal case** — if n equals the list length, you must return `head.next`, not attempt to navigate before it.
- **Off-by-one errors in the loop** — forgetting to move to `length - n - 1` instead of `length - n` in the two-pass approach.
- **Not using a dummy node** — the dummy node elegantly avoids the head case without extra branching; omitting it often leads to bugs.
- **Breaking the loop early** — in the two-pointer approach, iterating `n` times instead of `n+1` times will misalign the pointers.
- **Null pointer dereferences** — always check that the node and its next pointer exist before accessing them.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why two pointers over two pass?** The single-pass solution is more elegant and shows mastery of pointer manipulation. Interviewers often explicitly ask for one-pass solutions.
- **Why the dummy node?** It eliminates the need for a special case when removing the head. Discuss how dummy nodes simplify edge cases in linked list operations.
- **Gap maintenance:** The key insight is that by moving fast `n+1` steps ahead, we naturally create a gap of exactly `n` nodes. This gap is preserved as both pointers move together.
- **Edge cases to mention:** single-node lists, removing the head, removing the tail, and very small lists (n=1, length=1 vs. length=2).
- **Follow-up — General Node Removal:** Can extend to remove all occurrences of a value, or remove every kth node. The dummy node pattern is reusable.
</Aside>

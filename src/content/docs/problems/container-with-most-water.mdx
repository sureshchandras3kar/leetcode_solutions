## Problem Statement

You are given an integer array `height` of length `n`. There are `n` vertical lines drawn such that the two endpoints of the `i`th line are `(i, 0)` and `(i, height[i])`. Find two lines that together with the x-axis form a container, such that the container contains the most water. Return the maximum amount of water a container can store. Notice that you may not slant the container.

<Aside type="tip" title="Key formulas to remember">
- Area = min(height[left], height[right]) × (right - left)
- The minimum height is the bottleneck — water can only be as tall as the shorter wall
- Key greedy insight: if we move the taller pointer inward, width decreases AND the height stays the same or worse → always move the shorter pointer
</Aside>

### Examples

| Input | Output |
| --- | --- |
| `[1,8,6,2,5,4,8,3,7]` | `49` (between index 1 and 8: min(8,7)\*7=49) |
| `[1,1]` | `1` |

## Constraints

- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Water reservoir design** — choose two dam heights to maximize water storage
- **Histogram max rectangle** — similar greedy reasoning (see #84)
- **Antenna placement** — maximize signal reception area between two towers
- **Profit optimization** — maximize revenue = min(supply, demand) × time_gap
</Aside>

## Approach 1: Two Pointers (Recommended)

<ApproachTag tag="recommended" />

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" />

Start with the widest possible container — left pointer at index 0, right pointer at index n-1. At each step, compute the current area. Then move the pointer pointing to the shorter wall inward; keeping the taller pointer would only decrease width without any chance of a taller container.

**Greedy proof:** Suppose `height[left] < height[right]`. Moving `right` inward:
- Width decreases by 1
- New height = min(height[left], height[new_right]) ≤ height[left] — the left wall still caps the height
- So area can only decrease or stay the same → moving the taller pointer is never optimal
- Therefore, always move the SHORTER pointer

### Pseudocode

```plaintext
function maxArea(height):
    left, right = 0, len(height) - 1
    max_water = 0
    while left < right:
        water = min(height[left], height[right]) * (right - left)
        max_water = max(max_water, water)
        if height[left] <= height[right]:
            left++
        else:
            right--
    return max_water
```

## Approach 2: Brute Force

<ApproachTag tag="simple" />

<ComplexityBadges time="O(n²)" space="O(1)" timeColor="red" spaceColor="green" />

Check every possible pair of lines using two nested loops. For each pair `(i, j)`, compute `area = min(height[i], height[j]) * (j - i)` and track the maximum. Simple to understand but too slow for large inputs.

### Step-by-step Example

**Input:** `height = [1,8,6,2,5,4,8,3,7]`

| i | j | h[i] | h[j] | width | area | max |
| --- | --- | --- | --- | --- | --- | --- |
| 0 | 1 | 1 | 8 | 1 | 1 | 1 |
| 0 | 2 | 1 | 6 | 2 | 2 | 2 |
| ... | ... | ... | ... | ... | ... | ... |
| 1 | 8 | 8 | 7 | 7 | 49 | 49 |
| ... | ... | ... | ... | ... | ... | ... |

### Pseudocode

```plaintext
function maxAreaBruteForce(height):
    n = len(height)
    max_water = 0
    for i in range(n):
        for j in range(i + 1, n):
            water = min(height[i], height[j]) * (j - i)
            max_water = max(max_water, water)
    return max_water
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Moving the wrong pointer — always move the pointer with the **shorter** height, not the taller one
- Using `(j - i)` as the height instead of `min(height[i], height[j])` — the width is `j - i`, not the height
- Forgetting that water height is limited by the SHORTER of the two walls
</Aside>

## Interview Tips

<Aside type="tip" title="Key points to discuss">
- State the greedy proof immediately: "Moving the taller pointer can never improve the area — the shorter wall is always the bottleneck"
- This is O(n) which is optimal — any algorithm must consider each line at least once
- Common follow-up: "Can you prove the two-pointer never misses the optimal pair?" → yes, because we only discard pairs that are guaranteed suboptimal (any pair involving the shorter pointer with anything between the pointers produces a smaller area)

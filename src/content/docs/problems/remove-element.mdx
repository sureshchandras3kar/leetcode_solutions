## Problem Statement

Given an integer array `nums` and an integer `val`, remove all occurrences of `val` **in-place**. The order of the elements may be changed. Then return the number of elements in `nums` which are not equal to `val`.

Consider the number of elements in `nums` which are not equal to `val` be `k`, then you need to do the following things to be accepted:

- Change the first `k` elements of `nums` such that they contain all the elements which are not equal to `val`.
- The remaining elements of `nums` are not important as well as the size of `nums`.
- Return `k`.

<Aside type="tip" title="What to notice first">
- You must modify the array **in-place** without using extra space for another array.
- The problem only cares about the first `k` elements where `k` is the number of remaining elements.
- Elements after position `k` can be anything — they are ignored in validation.
- The most efficient solution uses **two pointers** to partition the array with a single pass.
</Aside>

### Examples

| Input | val | Output | Array After | Explanation |
| --- | --- | --- | --- | --- |
| `[3,2,2,3]` | `3` | `2` | `[2,2,_,_]` | The first 2 elements are `[2, 2]`; rest don't matter |
| `[0,1,2,2,3,0,4,2]` | `2` | `5` | `[0,1,4,0,3,_,_,_]` | The first 5 elements contain no `2` |
| `[2]` | `3` | `1` | `[2]` | No element equals `3` |

## Constraints

- `0 <= nums.length <= 100`
- `0 <= nums[i] <= 50`
- `0 <= val <= 100`

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Log filtering** — remove all entries with a specific error code in-place to compact log buffers.
- **Inventory cleanup** — remove all items of a discontinued SKU from a stock array without allocating new memory.
- **Packet filtering** — discard all packets of a specific type during stream processing on memory-constrained devices.
- **User session invalidation** — remove all sessions of a particular user state from an active session list.
</Aside>

## Approach 1: Two Pointers (Recommended)

<ApproachTag tag="recommended" />

Use two pointers: `read` scans the array, and `write` tracks where to place non-val elements. When `read` finds a value that is not `val`, copy it to the `write` position and increment both pointers. The first `write` elements will contain all non-val values.

The key insight is that we partition the array into two regions: the first `write` elements are valid, and the rest are ignored.

<ComplexityBadges time="O(n)" space="O(1)" timeColor="green" spaceColor="green" timeNote="Single pass" spaceNote="No extra data structures" />

### Step-by-step Example

**Input:** `nums = [0,1,2,2,3,0,4,2]`, `val = 2`

| Step | read | write | nums[read] | Action | Array State |
| --- | --- | --- | --- | --- | --- |
| 0 | 0 | 0 | 0 | 0 ≠ 2 → copy to write | `[0,1,2,2,3,0,4,2]`, write++ |
| 1 | 1 | 1 | 1 | 1 ≠ 2 → copy to write | `[0,1,2,2,3,0,4,2]`, write++ |
| 2 | 2 | 2 | 2 | 2 == 2 → skip | `[0,1,2,2,3,0,4,2]`, read++ only |
| 3 | 3 | 2 | 2 | 2 == 2 → skip | `[0,1,2,2,3,0,4,2]`, read++ only |
| 4 | 4 | 2 | 3 | 3 ≠ 2 → copy to write | `[0,1,3,2,3,0,4,2]`, write++ |
| 5 | 5 | 3 | 0 | 0 ≠ 2 → copy to write | `[0,1,3,0,3,0,4,2]`, write++ |
| 6 | 6 | 4 | 4 | 4 ≠ 2 → copy to write | `[0,1,3,0,4,0,4,2]`, write++ |
| 7 | 7 | 5 | 2 | 2 == 2 → skip | Done, return 5 |

Final result: First 5 elements `[0, 1, 3, 0, 4]` contain no `2`.

### Pseudocode

```plaintext
function removeElement(nums, val):
    write = 0
    for read in range(len(nums)):
        if nums[read] != val:
            nums[write] = nums[read]
            write++
    return write
```

## Approach 2: Brute Force

<ApproachTag tag="simple" />

Scan the array from the beginning. When you find an element equal to `val`, remove it by shifting all subsequent elements left by one position. Repeat until no more elements equal `val` are found.

This approach is simple to understand but is inefficient because each removal triggers a shift operation that takes O(n) time.

<ComplexityBadges time="O(n²)" space="O(1)" timeColor="red" spaceColor="green" timeNote="Shift per removal" spaceNote="No extra data structures" />

### Step-by-step Example

**Input:** `nums = [3,2,2,3]`, `val = 3`

| Step | i | nums[i] | Action | Array State | Explanation |
| --- | --- | --- | --- | --- | --- |
| 0 | 0 | 3 | 3 == 3 → remove | `[2,2,3]` | Shift elements left, length 3 |
| 1 | 0 | 2 | 2 ≠ 3 → skip | `[2,2,3]` | Move to next element |
| 2 | 1 | 2 | 2 ≠ 3 → skip | `[2,2,3]` | Move to next element |
| 3 | 2 | 3 | 3 == 3 → remove | `[2,2]` | Shift elements left, length 2 |
| Done | — | — | — | `[2,2]` | Return 2 |

Final result: First 2 elements are `[2, 2]`.

### Pseudocode

```plaintext
function removeElementBruteForce(nums, val):
    i = 0
    while i < len(nums):
        if nums[i] == val:
            remove nums[i]  // Shift all elements after i one position left
        else:
            i++
    return len(nums)
```

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting that you only need to return the count, not actually resize the array. Elements after position `k` can be anything.
- In the brute force approach, decrementing the index after removal — you should not increment in the else block, so adjust carefully.
- Moving only the write pointer when you find a non-val element — you must also advance the read pointer to keep scanning.
- Off-by-one errors when copying elements in the two-pointer approach.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Why two-pointer over brute force?** Trading one pass for multiple passes. The two-pointer approach is O(n) because each element is visited once; brute force is O(n²) because each removal triggers a shift.
- **Why not use a queue or list?** Using built-in remove operations often hides the cost — the problem specifically asks for in-place modification to make you think about the algorithm.
- **Edge cases to mention:** empty array (return 0), array with all elements equal to val (return 0), array with no elements equal to val (return the original length), single element (return 0 or 1 depending on val).
- **Follow-up — duplicate values?** The problem #26 extends this to handle duplicates with constraints on how many you keep.
</Aside>

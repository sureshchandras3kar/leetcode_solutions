## Problem Statement

Given an `n × n` matrix of `0`s and `1`s, construct a quad tree with the following rules:

1. If all values in a region are the same, create a leaf node with that value.
2. Otherwise, subdivide the region into 4 equal quadrants (top-left, top-right, bottom-left, bottom-right) and recursively build the tree.

<Aside type="tip" title="What to notice first">
- Pure divide and conquer: divide into 4 quadrants recursively.
- Base case: all values in a quadrant are the same → leaf node.
- Recursive case: quadrant has mixed values → subdivide further.
- Early termination when homogeneous regions are found.
</Aside>

### Examples

| Input | Output |
| --- | --- |
| `[[1,1],[1,0]]` | Quad tree with non-leaf root, mixed children |
| `[[1,1],[1,1]]` | Single leaf node with val=true |

## Constraints

- `n == 2^x` where `0 <= x <= 6`
- `grid[i][j]` is either 0 or 1

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- File system hierarchies and directory structures
- DOM tree manipulation in web browsers
- XML/JSON parsing and configuration management
- Organizational hierarchies and permission trees
</Aside>

## Complexity Comparison

| Approach | Time | Space |
| --- | --- | --- |
| Recursive | O(n² log n) | O(log n) |
| Iterative | O(n² log n) | O(n² / 4) |

## Approach 1: Recursive Divide and Conquer (Recommended)

<ApproachTag tag="recommended" />

1. Check if all values in the current region are the same.
2. If yes, create a leaf node.
3. If no, create a non-leaf node and recursively subdivide into 4 quadrants.

## Common mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Incorrect homogeneity check (must check entire region, not just corners).
- Wrong quadrant assignment (remember row-major order).
- Off-by-one errors in boundary calculations.
- Not handling the case where a region is already homogeneous (leaf node).
- Forgetting to set children pointers for non-leaf nodes.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **Homogeneity check:** O(size²) per region; memoization or prefix sums could optimize.
- **Recursive vs. iterative:** Recursive is cleaner; iterative uses more space but avoids stack overhead.
- **Applications:** Image compression (store regions of uniform color), spatial indexing, collision detection in games.
- **Follow-up:** How would you serialize/deserialize the quad tree?
- **Edge cases:** Single cell (always leaf), all zeros/ones (single leaf), all different values (deep tree).
</Aside>

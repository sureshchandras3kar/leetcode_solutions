## Problem Statement

Given two binary strings `a` and `b`, return their sum as a binary string.

You may assume neither string has leading zeros, except for the zero itself.

<Aside type="tip" title="What to notice first">
- Binary addition works exactly like decimal addition, just with carries at each power of 2.
- Process from right to left, tracking the carry bit just like in elementary school addition.
- The result string may be one bit longer than the inputs if there's a final carry.
</Aside>

### Examples

| Input a | Input b | Output | Explanation |
| --- | --- | --- | --- |
| `"11"` | `"1"` | `"100"` | `3 + 1 = 4` in binary |
| `"1010"` | `"1011"` | `"10101"` | `10 + 11 = 21` in binary |
| `"0"` | `"0"` | `"0"` | Edge case: zero + zero |

## Constraints

- `1 <= a.length, b.length <= 10^4`
- `a` and `b` consist of only `'0'` or `'1'` characters
- Each string does not have leading zeros except for the zero itself

## Real-World Applications

<Aside type="note" title="Where this pattern appears">
- **Arbitrary precision arithmetic** — implementing addition for numbers too large for native integer types.
- **Cryptographic operations** — multi-precision arithmetic in public-key algorithms.
- **Quantum computing simulators** — representing quantum state vectors as binary values.
- **Hardware design** — ALU (Arithmetic Logic Unit) simulation and validation.
</Aside>

## Complexity Comparison

| Approach | Time | Space | Notes |
| --- | --- | --- | --- |
| String Simulation | O(max(n, m)) | O(max(n, m)) | Manual carry propagation, easy to understand |
| Bit Operations | O(max(n, m)) | O(max(n, m)) | Convert to integers, add natively, convert back |

\* n = length of string a, m = length of string b

## Which approach should you learn first?

- Pick **String Simulation** if you want to understand how binary addition works at the bit level.
- Pick **Bit Operations** if you want the simplest implementation using built-in radix conversion.

## Approach 1: String Simulation (Recommended)

<ApproachTag tag="recommended" />

Simulate elementary school binary addition: process digits from right to left, maintaining a carry bit. When you reach the end of the shorter string, pad with zeros and continue.

<ComplexityBadges time="O(max(n, m))" space="O(max(n, m))" timeColor="green" spaceColor="yellow" timeNote="Single pass right to left" spaceNote="Result string storage" />

### Step-by-step Example

**Input:** `a = "1010"`, `b = "1011"`

| Step | i | j | a[i] | b[j] | carry | sum | Action |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 3 | 3 | 0 | 1 | 0 | 1 | Append 1, carry=0 |
| 2 | 2 | 2 | 1 | 1 | 0 | 2 | Append 0, carry=1 |
| 3 | 1 | 1 | 0 | 0 | 1 | 1 | Append 1, carry=0 |
| 4 | 0 | 0 | 1 | 1 | 0 | 2 | Append 0, carry=1 |
| 5 | -1 | -1 | - | - | 1 | - | Append 1, done |
| **Result:** `"10101"` |

### Bit-Level Visualization

```
    1010     (input a)
  + 1011     (input b)
  ------

Step 1:  0 + 1 + 0 = 1, no carry
Step 2:  1 + 1 + 0 = 10 (binary) = 0 with carry 1
Step 3:  0 + 0 + 1 = 1, no carry
Step 4:  1 + 1 + 0 = 10 (binary) = 0 with carry 1
Step 5:       carry 1 left = 1

Result: 10101
```

### Pseudocode

```plaintext
function addBinaryStringSimulation(a, b):
    result = []
    carry = 0
    i = len(a) - 1
    j = len(b) - 1

    while i >= 0 or j >= 0 or carry:
        digitA = a[i] if i >= 0 else 0
        digitB = b[j] if j >= 0 else 0
        total = digitA + digitB + carry
        result.append(total % 2)
        carry = total // 2
        i -= 1
        j -= 1

    return reverse(result).join('')
```

## Approach 2: Bit Operations

<ApproachTag tag="simple" />

Convert both binary strings to integers using the built-in radix parser, add them using native arithmetic, then convert the result back to binary.

This approach leverages language-native support for arbitrary precision (Python, JavaScript BigInt) or fixed-width integers with overflow-safe addition.

<ComplexityBadges time="O(max(n, m))" space="O(max(n, m))" timeColor="green" spaceColor="yellow" timeNote="Conversion + addition" spaceNote="Result string storage" />

### Step-by-step Example

**Input:** `a = "11"`, `b = "1"`

```
Step 1: int('11', 2) = 3
Step 2: int('1', 2) = 1
Step 3: 3 + 1 = 4
Step 4: bin(4) = '0b100'
Result: '100' (strip '0b' prefix)
```

### Pseudocode

```plaintext
function addBinaryBitOperations(a, b):
    numA = parseInt(a, 2)
    numB = parseInt(b, 2)
    total = numA + numB
    return bin(total).replace('0b', '')
```

## Common Mistakes

<Aside type="caution" title="Watch for these pitfalls">
- Forgetting to append the final carry bit if there's an overflow.
- Processing from left to right instead of right to left (carry won't propagate correctly).
- Confusing the output: you're building the result in reverse, so remember to reverse it.
- For languages with fixed-width integers (C++, Java), watch out for overflow on very large inputs.
</Aside>

## Interview Tips

<Aside type="tip" title="Key trade-offs to discuss">
- **String Simulation vs. Bit Operations:** String Simulation shows you understand binary arithmetic deeply; Bit Operations is faster to code in an interview but may seem like you're "cheating" with built-in functions.
- **Edge cases:** Zero strings, single-bit inputs, very long inputs (10^4 characters), carry overflow.
- **Follow-up:** Can you implement this without converting to integers (in languages with limited integer size)? Can you do it iteratively vs. recursively?
- **Why binary addition:** It's the foundation of CPU arithmetic; understanding this helps with hardware-level thinking.
</Aside>
